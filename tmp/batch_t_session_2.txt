# Workflow (Opus 1M — bulk-coherent per session)
Read all 80 candidate RVAs first; then decomp them in clusters of ~10-15 at a
time. Opus 1M context lets you keep 80 functions' decomp + cross-refs in scope
simultaneously — exploit this for cross-function naming (recurring callees,
shared globals, repeated string patterns identify clusters).

Per RVA:
  1. mcp__ghidra__function_at <rva>     (or listing_code_unit_at if no fn)
  2. mcp__ghidra__decomp_function       (the body)
  3. mcp__ghidra__function_callees      (depth-1 callees)
  4. mcp__ghidra__function_callers      (subsystem inference)
  5. Cross-ref the RW catalog: if the function's signature/calls/constants
     match an RW API entry, name accordingly and cite catalog row.
  6. Write re/analysis/bucket_0048ad50/<rva>.md with:
     - frontmatter (rva, name, size_bytes, confidence=C1, callees_depth1,
       callers_noted, opened_in_slot=Mashed_pool2, session_date=2026-05-18)
     - ## Mechanical description (bullets; no speculation; cite addresses)
     - ## Constants (table; cite RVA for each)
     - ## Uncertainties (file U-IDs to UNCERTAINTIES.md)
     - ## Stubs encountered (file S-IDs to STUBS.md)
