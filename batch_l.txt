   vehicle-physics callee. Across batches H-K we have ~50 vehicle/physics

   across game_state_d2 / d5 plates.

   chain (DAT_006XXXXX cluster).

6. **txd_texture** — drafted in batch_h Session 47; verify and extend.

# Inputs
- ALL per-RVA .md files in re/analysis/* (especially recent batch_i/j/k
  bucket dirs). Grep them for DAT_ references and offset citations.
- Existing struct files in re/analysis/structs/ — amend, don't rewrite.
- ROADMAP.md § "S-DoD" for the field-by-field RVA citation requirement.

# Workflow
1. Acquire $SLOT=Mashed_pool7.
2. Read re/analysis/structs/REPORT_phase5.md (if it exists from batch_h
   Session 48) to find what's missing.
3. For each target struct:
   a. Grep all per-RVA .md files for the DAT_ root candidate.
   b. Cluster offsets/widths/signs; cross-validate across ≥3 callers.
   c. Document offset, width, sign, first read RVA + cite, first write
      RVA + cite, tentative semantic name (only if multiple callers
      agree), [UNCERTAIN U-NNNN] if disagreement.
   d. Write or amend re/analysis/structs/<struct_name>.md.
4. NO hooks.csv mutations. NO scribe-queue row (no master writes for
   struct docs — they're git-tracked .md files, not Ghidra plates).
5. Commit at end.

# Output
- ≥3 new or amended struct files in re/analysis/structs/
- Updated re/analysis/structs/REPORT_phase5.md summarizing:
  - struct → DAT_ root → field count → confidence level
  - which subsystems' S-DoD this unblocks
  - any U-IDs filed for unresolved fields

# STOP-AND-ASK if
- A struct has fewer than 3 cross-references in plates — defer with a
  re-pickup condition ("after subsystem reaches N C1 rows").
- The same DAT_ root is referenced as DIFFERENT struct shapes from
  different subsystems — union or aliased global; design decision needed.
