================================================================================
BATCH G — C1 → C2 promotions in dense buckets
================================================================================

Six parallel sessions. Pool slots pre-assigned. Run after Batch F's scribe
sweep has drained.

This batch promotes the densest C1 clusters to C2. C2 promotion is the
gate for S-DoD (subsystem Definition of Done) per ROADMAP.md — every C2
plate needs an end-to-end body read with struct accesses tabled.

Bucket selection rationale: each session targets a 64K RVA bucket where
≥10 C1 rows live within the same logical cluster, so the work has natural
locality and the resulting C2 plates cross-reference cleanly.

All sessions follow re/SESSION_RULES.md.



================================================================================
Session 37 — render_promote_c2_track_loader  (RE-CLASSIFY, 0x00426 bucket)
================================================================================

Pool slot:   Mashed_pool0 (pre-assigned)
Model:       Sonnet 4.6 — hard cap 20 RVAs
Skill used:  re-classify
Bucket dir:  re/analysis/render_promote_c2_track_loader/

# Mission
The 0x00426xxx RVA range holds 22 render C1 rows, all in the track_loader
cluster (TRACK_LOAD_FN family, COURSE.LUA / LAPDATA.LUA / AI-data load
paths, path-builders, record processors). Promote the first 20 to C2.

# Inputs
- Filter hooks.csv: `subsystem=render, confidence=C1, RVA in
  [0x00426000, 0x00426fff]`. Take the first 20 by RVA.
- Existing C1 plates in re/analysis/track_loader/, track_loader_d3/,
  track_world_initial_sweep/, render_frame_d3/ — read first to avoid
  rework.

# Workflow
1. Startup with $SLOT=Mashed_pool0.
2. For each RVA:
   a. function_at + decomp_function — read body end-to-end.
   b. Document the C2 plate per re/CONFIDENCE.md § C2: signature, args,
      return, struct accesses table, callees with their confidence levels.
   c. Track the track-loading state struct via offsets seen across rows
      (DAT_0066d704, DAT_00644370, DAT_008991e0 were named in existing
      plates) and cross-link to track_collision_geometry/bsp_struct_a.md.
   d. [UNCERTAIN] for any field with disagreement across callsites.
3. Run re-classify per row. Reject promotions where the no-islands rule
   fails (at least one caller AND one callee must be at C2+).
4. If the track-loader state struct emerges clean, draft
   re/analysis/structs/track_loader_state.md.
5. Shutdown + scribe-queue append.

# Output
- 20 per-RVA .md files in re/analysis/render_promote_c2_track_loader/
- 20 hooks.csv promotions C1 → C2
- Possibly re/analysis/structs/track_loader_state.md (initial draft)
- Queued row in SCRIBE_QUEUE.md

# STOP-AND-ASK if
- More than 5 rows fail the no-islands rule (their callees are all C0/C1).
  Decide whether to map the callees first (different session) or pick a
  subset of the 22 with cleaner cluster boundaries.
