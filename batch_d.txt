Session 21 — ai_powerup_decisions  (INVESTIGATION → first-pass)
================================================================================

Pool slot:   Mashed_pool4 (rotated from Batch A)
Model:       Sonnet 4.6 — hard cap 20 RVAs (sweep phase)
Bucket dir:  re/analysis/ai_powerup_decisions/

# Mission
Map the AI side of powerup-use decisions. Mashed is a combat racer — AI
drivers shoot powerups at the player and use defensive powerups. Find the
code that decides WHEN AND WHICH.

# Decision protocol — entry-point discovery FIRST
1. Read re/analysis/powerups_d3 / powerups_d4 / powerups_d4_repair plates
   (the player-side powerup dispatch is largely mapped). Look for callers
   that aren't the player input pipeline.
2. The AI side should call into the SAME powerup-fire/use functions, but
   driven by AI logic instead of button presses. Find the AI callers.
3. xrefs_to on the powerup-fire entry points from powerups_d3/d4 plates —
   filter out callers in subsystem=input.
4. Candidate entry points: the non-input callers of powerup-fire.
5. If ≥1 candidate → PROCEED.
6. If nothing → file as DEFERRED; AI may use a separate event/queue system
   that needs to be discovered first.

# Sweep path
7. Startup with $SLOT=Mashed_pool4.
8. From each candidate: callees_depth1. Cap 20.
9. Per-RVA C1 plates.
10. Document the AI decision parameters (distance to target, target's
    facing, current powerup inventory, AI difficulty level).
11. Shutdown + scribe-queue append.

# Output
- EITHER: closure + DEFERRED row
- OR: ≤20 per-RVA .md files + ≤20 C1 hooks.csv rows + Queued row

# STOP-AND-ASK if
- The xrefs reveal AI uses a different powerup-fire path than the player
  (different fire-function entry point). Confirm whether that's a separate
  subsystem to map.
