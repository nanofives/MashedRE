## Subset definition (anchor-driven, halt-allowed)
SciLor's MashedRunner documents that launch.exe (VB6 wrapper) performs a
copy-protection check and then spawns MASHED.exe with specific CLI args.
This session looks for the **MASHED.exe-side** of that handshake — what
arguments / IPC / registry keys MASHED.exe consumes from launch.exe, and
whether a protection check is duplicated inside MASHED.exe.

This work is for the GREENFIELD port: identify what mashed_re.exe must
emulate from launch.exe so it can run standalone without launch.exe.

Strategy 1 — CLI / argv anchors:
  search_defined_strings query="-language"
  search_defined_strings query="-lang"
  search_defined_strings query="-window"
  search_defined_strings query="-fullscreen"
  search_defined_strings query="-nointro"
  search_defined_strings query="-nosplash"
  search_defined_strings query="-debug"
  external_imports_list — look for GetCommandLineA/W, CommandLineToArgvW.

Strategy 2 — registry anchors:
  search_defined_strings query="Software\\Empire"
  search_defined_strings query="Software\\Strategy First"
  search_defined_strings query="Software\\Supersonic"
  search_defined_strings query="Software\\Mashed"
  search_defined_strings query="HKEY_"
  external_imports_list — RegOpenKeyExA/W, RegQueryValueExA/W.

Strategy 3 — copy-protection signatures:
  search_defined_strings query="StarForce"     (period CD-ROM protection)
  search_defined_strings query="SafeDisc"
  search_defined_strings query="securom"
  search_defined_strings query="SECDRV"
  search_defined_strings query="cdcheck"
  search_defined_strings query="DVD"
  search_defined_strings query="please insert"
  external_imports_list — GetVolumeInformationA/W, GetDriveTypeA/W,
  GetLogicalDrives, CreateFileA on \\.\CdRom*.

Strategy 4 — IPC / mutex:
  external_imports_list — CreateMutexA/W, OpenMutexA/W, CreateFileMappingA/W.
  Mutexes named "MashedSingleInstance" or similar would be the
  launch.exe-side handshake anchor.

Strategy 5 — entry-point scan:
  IF entry_callees bucket exists, the WinMain or _start callees include
  argv parsing and any early-exit copy-protection check. Read entry_callees
  notes for branches that exit before the main game loop.

Identify HANDSHAKE_FN: function(s) that consume launch.exe's contract.

If NONE converges OR all anchors absent: halt + surface "launch.exe handshake anchors absent in MASHED.exe — protection may live entirely in launch.exe (PE-level disc check), with MASHED.exe consuming only language CLI args. Halting cleanly. (This is a useful greenfield finding: mashed_re.exe needs only argv-language consumption, no protection bypass.)"

If found: function_callees of the relevant handshake function(s) → subset capped 20/15. Also verify CLI param parsing: which exact strings does MASHED.exe accept, what registry keys does it read, what mutex (if any) does it create.
