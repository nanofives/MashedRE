## Overview

Four **physics body creator** functions, all sharing the same 6-argument signature and same registration loop. Differ only in shape-setup call.

```c
void FUN_0047fXXX(int count, undefined4 param_2, undefined4 world, undefined4 scene,
                  int obj_idx, undefined4 clump_or_dff);
```

- count: number of body instances to create
- world: physics world handle (param_3 used in FUN_0055dec0)
- scene: scene parent handle (param_4 used in FUN_004e4380)
- obj_idx: index into scene-object table at DAT_0086cbcc; stride 0x44 dwords (0x110 bytes)
- clump_or_dff: DFF/clump handle for mesh data

## Common registration loop (per instance)

1. `FUN_0057c300(shape_desc)` — create shape instance → iVar
2. `DAT_006c6b90[DAT_006c6eb0] = obj_table[obj_idx].base_id + iter` — set polygon ID
3. `DAT_006c9758[DAT_006c6eb0] = obj_idx` — store object index
4. `DAT_006c6eb8[slot] = 1` — mark as box/sphere/etc. (box-type flag, only in fc40/fe00/ff70)
5. `FUN_004e69a0(clump)` → body handle → `DAT_006c71d8[slot]`
6. `FUN_004c0b30()` → transform → `FUN_004e7e30(body, transform)` — attach transform
7. `FUN_0057c220(body, shape_inst)` — bind shape
8. `FUN_004e4380(scene, body)` — add to scene
9. `*(iVar+8) &= 0xfffffff3` — clear collision type bits
10. `FUN_0055dec0(world, iVar, -1, 2)` + `FUN_00559c40` — add to world with mode 2
11. `DAT_006c686c[slot] = radius² from obj_table` — store squared radius for broad-phase
12. `DAT_006c6eb0++`

Shape descriptor lifecycle: `FUN_0057c500()` → setup → loop → `FUN_0057c550()`.
