## Body summary (0x00552d10–0x00552d66)

Saves the current font context matrix by pushing a new entry onto the font matrix stack.

1. **0x00552d10** `bVar1 = (DAT_00912b04 < 0x1f)` — overflow check; max 31 levels (0x1f)
2. If `bVar1 == false`: return false immediately
3. **0x00552d1c** `DAT_00912b04 = DAT_00912b04 + 1` — increment stack depth
4. **0x00552d25** If `(&DAT_00912a84)[DAT_00912b04] == 0`: lazy-allocate new slot
   - `(&DAT_00912a84)[DAT_00912b04] = FUN_004c57a0()` (**0x00552d2e**)
5. **Copy previous context into new slot (64 bytes = 16 dwords)**:
   - `puVar4 = *(undefined4**)(&DAT_00912a80 + DAT_00912b04 * 4)` — ptr to previous ctx (slot[new-1])
   - `puVar5 = (&DAT_00912a84)[DAT_00912b04]` — ptr to new ctx
   - 16-iteration copy loop at 0x00552d41–0x00552d5e
6. **0x00552d63** return `bVar1` (true)

### Array pointer arithmetic note:
`&DAT_00912a80 + new_idx * 4` with new_idx incremented addresses `&DAT_00912a84 + (new_idx-1)*4` = `font_ctxs[new_idx - 1]`. This is the previous top of stack. DAT_00912a80 is 4 bytes before DAT_00912a84.
