## Mechanical description

Per-player joypad state update. Player index arrives in unaff_EBX (register convention, not stack param).

- stride = unaff_EBX * 0x4c (76 bytes per player in DAT_007f14f8 array).
- Copies 0x13 (19) entries from DAT_007f1038[player*0x13] to DAT_007f14f8[player*0x4c] (save previous state).
- Zeroes 0x13 entries in DAT_007f1038[player*0x13] (clear current state).
- If config flag DAT_007e96fc[player*0x80] == 1 (joystick mode): calls FUN_004957a0(DAT_007e9700[player*0x80], 0, &local_8, 0, &local_10).
- Calls FUN_00497310 nine times (button slots 0..8 via param_2) to read: btn3 index, btn2/btn3/btn1/btn5 bytes; up-axis; down-axis; and two axis keys.
  - Results written to: DAT_007f1042, 007f1043, 007f1041, 007f1049, 007f103c, 007f1077, 007f1040 (per-player offsets).
  - Button 0: cVar3 from FUN_00497310; if non-zero: sets DAT_007f1044=0xFF and writes key to DAT_007f1038.
  - Buttons 1..3 handled similarly.
- Axis mode 1 (joystick): copies local_8 (axis value) and negates local_4 for stick output.
- Axis mode 2 (button-as-axis): computes float axis from button differences * scale at _DAT_005ceb90.
- Swaps X/Y if DAT_007f0f30 != 0 (swap-axes flag).
- Player 1 special case (unaff_EBX == 1):
  - Saves/shifts DAT_008989c0/c4/a0/a4 pair [UNCERTAIN U-2304].
