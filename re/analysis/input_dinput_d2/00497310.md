## Mechanical description

Device-agnostic button/axis state query. param_1 = player/device index (0..3); param_2 = button slot index.

Device type is at DAT_007e96fc[param_1 * 0x80].

**Type 1 (joystick/gamepad path):**
- Reads float axis values fVar1, fVar2 from float arrays at 0x0077311c and 0x00773120 (stride = device_index * 8).
- For param_2 9..12 (analog axis comparisons):
  - 9: negative X axis (fVar1 < threshold DAT_005d757c) → calls FUN_004a2c48 → returns byte.
  - 10: positive X axis.
  - 11: positive Y axis (fVar2).
  - 12: negative Y axis.
- For all other param_2 (digital buttons):
  - Reads button index from DAT_007e96c8[param_1 * 0x80 + param_2].
  - Returns 0xFF if bit set in button bitmap at DAT_007730d4[device*4 + byte_index], 0x00 otherwise.

**Type 2 (keyboard path):**
- Reads mapped scancode from DAT_007e96c8[param_1 * 0x80 + param_2].
- Tests bit in keyboard bitmap at DAT_0077313c.
- Returns 0xFF if pressed, 0x00 otherwise.

**No valid device:** returns 0.
