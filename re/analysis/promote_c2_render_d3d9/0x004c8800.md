## What it does
Selects D3DPRESENT_PARAMETERS fields — back-buffer format, depth/stencil format, and MSAA level — for the forthcoming `CreateDevice` call.

### Windowed path (`param_2 & 1 == 0`)
- `DAT_00912100` = 1 (windowed flag)
- Calls `GetWindowRect(DAT_007d4104, &local_10)` — reads current window dimensions.
- Back-buffer `width`/`height` from RECT (0 if degenerate).
- Refresh rate `DAT_00912110` = 0; swap effect `DAT_009120f8` = 3 (D3DSWAPEFFECT_DISCARD).
- `uVar4 = param_3` (fallback format).

### Fullscreen path (`param_2 & 1 != 0`)
- `DAT_00912100` = 0
- Width `DAT_009120e0 = *param_1`, height `DAT_009120e4 = param_1[1]`.
- Refresh `DAT_00912110 = min(param_1[2], DAT_006181c4)`.
- `uVar4 = param_1[3]` (display mode format: D3DFORMAT).
- Swap effect `DAT_009120f8` = 2 (D3DSWAPEFFECT_FLIP).

### Back-buffer bit-depth classification (switch on `uVar1`):
| uVar1 (D3DFORMAT) | `DAT_00911f9c` | Bits |
|---|---|---|
| 0x15 (D3DFMT_R5G6B5), 0x16 (D3DFMT_X1R5G5B5), 0x23 | 0x20 | 32-bit |
| 0x17, 0x18, 0x19 | 0x10 | 16-bit |
| other | 0x00 | unknown |

Format `0x16→0x15` fallback via `IDirect3D9::CheckDeviceType` (vtable `+0x24`).

### Depth/stencil format selection (for 32-bit: vtable calls `+0x28` = CheckDeviceFormat, `+0x30` = CheckDepthStencilMatch):
- Try D3DFMT_D24S8 (0x4B) → D3DFMT_D24X8 (0x4F) → D3DFMT_D16 (0x47) → D3DFMT_D24X4S4 (0x4D) for 32-bit.
- Try D3DFMT_D15S1 (0x49) → fallback D3DFMT_D16 (0x50) for 16-bit.
- Selected format stored at `DAT_00912108`; stencil flag at `DAT_007d4124` (1 = has stencil).

### MSAA selection (tail):
- If MSAA supported (`DAT_006181b8 > 1` or `DAT_006181c0 > 1`) and not software rast:
  - Probes `CheckDeviceMultiSampleType` (vtable `+0x2c`) for the chosen MSAA level.
  - `DAT_009120f0` = selected MSAA type, `DAT_009120f4` = quality level − 1.
  - On success: `DAT_009120f8` = 1 (D3DSWAPEFFECT_DISCARD forced).
