## Mechanical description
- Cubic-spline keyframe interpolator/evaluator over a 2D array of keyframes held in `param_1` (animation-track container).
- Container layout (read via `int *param_1`):
  - `param_1[0]`  = base pointer to keyframe records (stride 0x30 each — same struct as 0x00489450/0x00489480/0x004894a0 fill into).
  - `param_1[2]`  = record count along axis A.
  - `param_1[3]`  = record count along axis B (outer loop variable `local_28`).
  - `param_1[0x14]` = bitmask of which channels to update (bits 0x02, 0x04, 0x10, 0x20 each gate a different field-emitter).
  - `param_1[0x17]` = float dt step.
  - `param_1[0x18]` = float current `t` (advanced by `param_1[0x19]` each call; wraps mod `param_1[0x17]`).
  - `param_1[0x19]` = float t-advance per frame.
  - `param_1 + 0x10` = RW frame/atomic handle pulled by `FUN_00476d00`.
- Per inner loop iteration, **cubic-polynomial evaluation** of the form
  `(2*p1 + (p2-p0)*t + (-p0 + p2 + 2*(p0+p3) - …)*t^3 + (…)*t^2) * (1/2)`
  on four control-point neighbours `pfVar12, pfVar15, pfVar13, pfVar9` per axis (x,y,z reads at offsets +0x00, +0x04, +0x08 of the record's vec3 field starting at base+4 of each record).
- Constants `_DAT_005cc31c, _DAT_005cc320, _DAT_005cc32c, _DAT_005cc358, _DAT_005cc35c` are pulled from the shared static-data pool (same pool used by D3DX9 inlined Hermite/Catmull-Rom — see _LIBRARY_RESIDUE_004ec000.md — but here they are used by **real game code** to build the same coefficients).
- Lerps a separate pair of attributes (4-vec at record offsets +0x10 and +0x40 (=0x10 of next record)) by `local_4c` and stuffs them into `local_1c`, `local_18`, `local_20`.
- Each channel-bit dispatch writes the result through one of:
  - `FUN_004769a0(&local_14)` — bit 0 of `param_1[0x14]` (always, vec3 emit)
  - `FUN_004769d0(&local_1c)` — bit 1 = 0x02 (vec2 lerped)
  - `FUN_004769f0(&local_20)` — bit 2 = 0x04 (single value emit)
  - `FUN_00476a30(*(undefined4 *)(...+0x18))` — bit 4 = 0x10
  - `FUN_00476a40(...+0x20)` — bit 5 = 0x20
- Finally calls `FUN_00476d00(param_1 + 4)` once per inner step — applies the accumulated transform to the RW node hosted at `param_1+0x10..+0x40`.
- Outer two nested loops iterate axis-A × axis-B keyframes (`param_1[3]` and `param_1[2]`) skipping records with `*(int *)(rec+0x1c) != 0` (the "marker / break" field).
