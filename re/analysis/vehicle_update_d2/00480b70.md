## Mechanistic description

Single-parameter function: `param_1` (int, vehicle index). Updates wheel-contact particle/
dust effects for the specified vehicle.

### Wheel contact counter

Reads 4 per-wheel contact flags at stride 0xd04 from vehicle-index base:
- `DAT_0088173c + param_1*0xd04` (wheel 0)
- `DAT_00881800 + param_1*0xd04` (wheel 1)
- `DAT_008818c4 + param_1*0xd04` (wheel 2)
- `DAT_00881988 + param_1*0xd04` (wheel 3)

Accumulates `cVar4` = count of non-zero wheel contact flags (0..4).

### Physics body retrieval

Calls `FUN_0057c210((&DAT_006c9a78)[param_1])` (0x0057c210) — gets RW physics body
pointer for this vehicle (`iVar2`).

Gets matrix base: `local_20 = *(int*)(*(int*)(iVar2+0x10) + 8)`.

### Vehicle wake check

Reads body active-set: checks bitset at `*(uint*)(*(int*)(*(int*)(iVar2+0x24)+0x60) + ...)`.
If body sleeping:
- Calls `FUN_0055ad30(*(uint*)(iVar2+0x24), iVar2)` — add to active-set.
- Calls `FUN_0055ac00(*(uint*)(iVar2+0x24), iVar2, 1)` — mark active.

### Transform snapshot

Calls `FUN_0046d4a0(&local_20, param_1)` (0x0046d4a0) — reads current vehicle frame
matrix into `local_20`.

### Particle emission (wheel-drive)

`iVar1 = *(int*)(*(int*)(iVar2+0x10) + 4)` — first linked body.
Checks `FUN_0046c7b0(param_1) != 0` (drive active) and `cVar4 != 0` (any wheel contact).

**Drive particles** (control block byte `*(char*)(puVar3 + 1) != 0`):
- Reads position from matrix `local_20 + 0x30/34/38` into `local_18/14/10`.
- Reads particle velocity: `(&DAT_00881f74)[param_1*0x341] * _DAT_005ccd6c` (X),
  `(&DAT_00881f78)[param_1*0x341] * _DAT_005ccd6c` (Y),
  `(&DAT_00881f7c)[param_1*0x341] * _DAT_005ccd6c` (Z).
- Calls `FUN_0055b650(iVar1, &velocity, &position)` (0x0055b650) — emit RW particle.

**Boost particles** (control block byte `*(char*)((int)puVar3 + 5) != 0`):
- Same position from matrix.
- Reads particle velocity with scale `_DAT_005cd61c` (boost-specific scale).
- Calls `FUN_0055b650(iVar1, &velocity, &position)` — emit RW particle.
