## Mechanical description

FUN_005bf050). `param_1` = filter; `param_2..param_7` = forwarded arguments. Returns the last
negative HRESULT, or 0.

For each pin in the array at `+0x11c` (count `+0x120`):
1. `pin = array[i]`; `hr = (**(pin + 0x24))(pin, &dir)` — vtable +0x24 (QueryDirection-shaped).
2. If `hr >= 0` and `dir == 1` (output) and `inner = *(pin + 0x114)` is non-null:
   - `hr = (**(inner + 0x44))(inner, param_2, param_3, param_4, param_5, param_6, param_7)` — vtable
     +0x44 with the six forwarded args.
   - If `hr < 0`: remember it.
3. Continue.
Return the remembered HRESULT (0 if none failed).

The 6-arg shape (e.g. NewSegment/Notify-style fan-out to output pins) parallels FUN_005bf050's no-arg
broadcast.
