## Mechanical description
Enumerator search/walk over a COM enumerator (IEnumPins / IEnumMediaTypes-shaped). `param_1` = the
object exposing the enumerator. Returns the matched item pointer (or a fallback object pointer). The
function body is heavily register-stitched (multiple `unaff_*` registers and an `iRam00000000`
artifact); only the mechanical shape is recoverable:

1. `hr = (**(*param_1 + 0x28))()` — vtable +0x28 (obtain enumerator). If `hr < 0` → return the
   fallback (`puStack_c`).
2. Reset the enumerator (`vtable +0x14`), then `enum->Next(1, &item)` (`vtable +0x0c`).
3. Loop over items: query each (`vtable +0x24`), compare to a target register value; on the matching
   index, break with the item; otherwise Release (`vtable +8`) and advance with another `Next`.
   When Next signals end, Release the cursor and return the accumulated result.
4. Releases the enumerator (`vtable +8`) on exit.
