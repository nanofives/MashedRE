## Mechanical description

Prints ridge data for one facet.

If `(facet+0x3c & 0x800) != 0 AND DAT_00914618 != 0` (visible-only & qh.NEWfacets):
  - Emits `"     - ridges (ids may be garbage):"`.
  - Walks f.neighbors at +0x24 emitting just the ridge ids (`*(ridge+0xc) & 0xffffff`).
  - Newline + return.

Else: emits `"   - ridges:"`. Clears `(ridge+0xc) & 0xfeffffff` (clear printed bit) for all neighbours of this facet.

Two paths based on `DAT_0091401c == 3` (hull_dim==3):
  - **3-D path**: walks ridge-chain via FUN_00581170 (qh_nextridge3d) per facet+0x24. Per ridge: ORs 0x1000000 (printed bit), emits `"     - r%d"`, optional flags `"tested"` / `"nonconvex"`, `"vertices: "`, per-vertex `"p%d(v%d)"`, then `"between f%d and f%d"`.
  - **N-D path**: nested walk of f.neighbors × f.ridges, matching pairs, same emit sequence.

After main walk: if `local_8 != FUN_005834a0(f.neighbors)` (count mismatch): emits `"   - all ridges:"` and dumps ids again.

Final walk through f.neighbors emitting any not-yet-printed ridges (`(byte+0xf & 1) == 0`).
