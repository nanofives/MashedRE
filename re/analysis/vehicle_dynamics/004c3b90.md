## Mechanical description

Fast reciprocal-sqrt (1/sqrt) via two-level lookup table. Paired with `FUN_004c3ac0` (vector
magnitude, sqrt variant) and uses the same shared table infrastructure at DAT_007d3FF8/007d3FFC.

### Algorithm

```c
float10 FUN_004c3b90(float param_1) {
    if (param_1 == 0.0f) return 0.0;
    uint bits = (uint)param_1;
    uint idx = ((bits + 0x800) >> 12) & 0xFFF;          // 12-bit table index from exponent+upper mantissa
    uint table_base = *(int*)(*(int*)(DAT_007d3ffc + 4 + DAT_007d3ff8));  // second-level ptr
    uint lut_val = *(int*)(table_base + idx * 4);
    uint mantissa_correction = (~(bits + 0x800) >> 1) & 0x3FC00000;      // bit-flip mantissa
    return (float)(lut_val + mantissa_correction);
}
```

Difference from FUN_004c3ac0 (sqrt):
- FUN_004c3ac0: `(bits + 0x800) >> 1 & 0x3FC00000` — add half exponent bits (sqrt)
- FUN_004c3b90: `(~(bits + 0x800) >> 1) & 0x3FC00000` — bitflip before shift (invert exponent = reciprocal)

### Lookup table

- First-level pointer: `DAT_007d3ff8` (base ptr)
- Second-level pointer: `DAT_007d3ffc` (offset +4 into ptr)
- Same table structure shared with FUN_004c3ac0.
- Index: upper 12 bits of (input_bits + 0x800) → 4096-entry table.

### Notes

- Returns `float10` (x87 extended precision).
- Zero guard returns 0.0 directly (avoids table miss at index 0).
- Used by FUN_0046ef70 (wheel contact resolver) to compute lateral force magnitude.

## Asm-level note (added during C3 promotion 2026-05-09)

Same caveat as `0x004c3ac0` Vec3Magnitude: Ghidra renders the final step
`(float)(lut_val + mantissa_correction)` as a numeric int→float cast, but
the asm at `0x004c3bdc..0x004c3be6` is:

```
ADD EAX, ECX            ; EAX = LUT_root_invsqrt[idx] + exp_correction (INT add)
MOV [ESP+4], EAX        ; spill
FLD float ptr [ESP+4]   ; reinterpret BITS as float, NOT int->float convert
RET
```

Verified semantics:

```
result = bit_cast<float>( LUT_root_invsqrt[idx] + ((~biased >> 1) & 0x3fc00000) )
```

LUT root reads at offset `+4` inside the RW globals (vs `+0` for the sqrt
LUT used by `0x004c3ac0`). The `NOT` before the right-shift inverts the
exponent contribution, producing the reciprocal sqrt instead of sqrt.

Reimplementation: `mashedmod/src/mashed_re/Math/RwSqrt.cpp` (`FastInvSqrt`).
