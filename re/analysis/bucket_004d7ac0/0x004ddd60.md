## Mechanical description
- Pack-4-bytes-to-D3D-format helper: writes a single 32-bit packed pixel into `*param_1` from the 4-byte BGRA-ish source `param_2` per format code in `param_3 & 0xf00`.
- Switch on `param_3 & 0xf00`:
  - `0x000`: recursion with `0x500` (raw-32 pack) then re-emit.
  - `0x100`: 16-bit BGRA1555-style (sign bit + 5/5/5)
  - `0x200`: 16-bit BGR565
  - `0x300`: 16-bit ARGB4444
  - `0x400`: 8-bit grayscale (Y = (0.30R + 0.59G + 0.11B) * A/255)
  - `0x500`: raw-32 pack (B<<24 | A<<16 | G<<8 | R) — but the `CONCAT31(CONCAT21(CONCAT11(...)))` form indicates an unusual byte interleave.
  - `0x600`: 24-bit packed BGR
  - `0xa00`: 16-bit BGR555-no-alpha
- On unknown format: pushes RWE error 0x8000000d via FUN_004d7ff0 / FUN_004d8480 and returns 0.
