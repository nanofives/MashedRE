## Mechanical description
- Computes the total in-memory byte cost of a D3D9 texture or cubemap, including all mip levels and metadata, into `*param_1`.
- Reads the texture handle from `*param_2`. If null: returns `0x54` (default constant ~ 84 bytes — header-only cost).
- Header cost base:
  - flag bit `0x40` at `+0x23` → 0xe4 (228) → likely cube-volume
  - flag bit `0x20` → 0x464 (1124) → likely volume / many faces
  - else → 0x64 (100) → plain texture
- Reads sub-handle pointer `*(int*)(DAT_00911ae4 + iVar4)`. Branches on the low nibble of `+0x9` byte (texture-type) and high nibble of `+0xa` byte (format/dxt):
  - Plain 2D (low nibble of +0x9 ≠ 0): calls vtable `+0x34` (GetLevelCount). For each level: GetSurfaceLevel `+0x44`, GetDesc/Lock `+0x4c`, ReleaseDesc/Unlock `+0x50`, sums `width * height` (or `width * height >> 2` if DXT — high nibble of +0xa nonzero with width>=4).
  - Cube map (low nibble of +0x9 = 0, sub-low-nibble ≠ 0): same six-times pattern (one per face).
- Writes `(header + sum_of_levels + 4_per_level)` into `*param_1`.
