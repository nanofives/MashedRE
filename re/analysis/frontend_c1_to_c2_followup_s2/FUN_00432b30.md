## Body Summary

Emits one or two HUD glyph slot entries into the stride-0xd table at `DAT_00898ac0` (via `*unaff_ESI` cursor stored in ESI).

**iVar3 resolution** (player-count / active-players index):
- `in_EAX == 0`: if `DAT_0067e9f8 != 0`, calls `FUN_0042add0()` for iVar3; if result == -1, resets to 0.
- `in_EAX == 2`: forces `param_1 = 0`, calls `FUN_0042add0()` for iVar3 unconditionally.
- Otherwise: increments `DAT_0067e9f8`, calls `FUN_0042add0()` for iVar3 (clamps -1 to 0), then decrements `DAT_0067e9f8`.

**Outer switch on `param_1`** (button/glyph type, cases 1/2/4/5/6/8/10):
- Each case calls `FUN_0042ad10(0x40, 0x1ac, 0)` to allocate/prepare a slot.
- Sets `(&DAT_00898ae8)[*unaff_ESI * 0xd]` to the primary glyph ID for the left button image.
- **Inner switch on `iVar3`** (player count, cases 1/2/4/5/6/8/10):
  - Sets `(&DAT_00898aec)[*unaff_ESI * 0xd]` to the secondary glyph ID.
  - Optionally writes `(&DAT_00898ac4)[*unaff_ESI * 0xd]` (anim state) and `(&DAT_00898ad0)[*unaff_ESI * 0xd]` (alpha/value).
  - Certain cases early-return; others fall through to common epilogue.

**Common epilogue** (`LAB_00432ed3`):
- Sets `(&DAT_00898ac4)[*unaff_ESI * 0xd] = 0`.
- Sets `(&DAT_00898ad0)[*unaff_ESI * 0xd] = 0x1ff`.
- Increments `*unaff_ESI`.

Special glyph IDs used: `0x42`, `0x43`, `0x225`, `0x48` / `0x13` (pair), `0x58`, `0x133`. `FUN_0042a9c0()` provides a dynamic glyph ID for cases 8 (controller-type lookup).  
For case 1 outer with `param_2 == FUN_0042b920()`, primary glyph is set to `-1` (0xffffffff).
