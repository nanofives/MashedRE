## Mechanical description

Wide-character printf formatting engine (`_woutput` CRT internal). Entry point of the MSVC CRT printf/wprintf call chain.

- Parameters: `(undefined4 param_1, byte* param_2, wchar_t* param_3)`
  - param_1: output context (passed through to write_char/write_multi_char/write_string)
  - param_2: format string (byte pointer, ASCII)
  - param_3: va_list (wchar_t* pointer to variadic args on stack)
- Stack-cookie guard via `__security_check_cookie`.
- Drives a two-table state machine: `DAT_005d1280` (character class table) and `DAT_005d12a0` (transition table). Each format byte is classified into class 0–7; transition selects which handler runs.

**Format specifiers handled (case 7 dispatch):**
| Specifier | Action |
|-----------|--------|
| `%d` / `%i` | signed decimal integer |
| `%u` | unsigned decimal |
| `%o` | octal (radix 8) |
| `%x` / `%X` | hexadecimal (lower/upper) |
| `%p` | pointer (8-hex-digit) |
| `%e` / `%E` / `%f` / `%g` / `%G` | float (via vtable PTR_FUN_006160d8) |
| `%s` / `%S` | narrow/wide string |
| `%c` / `%C` | narrow/wide char |
| `%n` | write count into arg |
| `%Z` | ANSI_STRING / UNICODE_STRING struct |

**Flags parsed (case 2):**
- 0x20 = space, 0x23 = `#` (alt-form), 0x2b = `+`, 0x2d = `-` (left-align), 0x30 = `0` (zero-pad)

**Length modifiers (case 6):**
- `h` = short (0x20), `l` = long (0x10), `w` = wide (0x800), `I64` = 64-bit (0x8000), `I32` = clear 64-bit

**64-bit path:** if `%I64d` etc., reads two stack words (QWORD) and uses `__aulldvrm` for division/modulo.

**Wide string / char:** uses `_wctomb` to convert each wide character to multibyte on output.
