# 0x00476df0 — ParticleBatch_Flush
**Confidence:** C2  
**Subsystem:** render  
**Status:** mapped

## Mechanical description

Flushes particle batch pool to GPU vertex buffers. No-op if `param_1[0xc] == 0` (empty batch).

### Flush sequence (one branch per non-null channel)

For each channel, if the channel array ptr is non-null:
1. Call `FUN_00535700(param_1[4], &lock_ptr, channel_id, 0x40000000)` — locks vertex buffer channel; 0x40000000 = D3DLOCK_DISCARD.
2. Copy `param_1[0xc]` entries from staging array into locked buffer.
3. (Unlock is deferred to `FUN_00535910` after all channels.)

After all channels:
- `FUN_00535910(param_1[4])` — unlock vertex buffer.
- `param_1[0xc] = 0` — reset count.

### Channel IDs passed to FUN_00535700
| Channel | ID | Stride | Data |
|---------|----|--------|------|
| pos_array (+0x10) | 1 | 0x0c (3 floats) | XYZ per particle |
| vel_array (+0x14) | 4 | 0x08 (2 floats) | XY per particle |
| colour_array (+0x18) | 2 | 0x04 | 1 dword |
| matrix_array (+0x1c) | 8 | 0x40 (16 dwords) | 4×4 matrix |
| scalar_array (+0x20) | 0x20 | 0x04 | 1 dword |
| rgba_array (+0x24) | 0x80 | 0x10 (4 dwords) | RGBA tuple |
| block_array (+0x28) | 0x100 | 0x20 (8 dwords) | block |

### Batch pool struct fields used
| Offset | Field |
|--------|-------|
| +0x04 | vertex buffer handle (passed to FUN_00535700, FUN_00535910) |
| +0x0c | entry count (set to 0 on exit) |
| +0x10..+0x28 | channel array ptrs (same layout as FUN_00476d00) |

### Additional write at entry
`*(param_1[4]+0x40) |= 0x800000` — sets a flag in the vertex buffer object.
`*(param_1[4]+0x04) = param_1[0xc]` — writes count into VB header.

### Callees
- `FUN_00535700` — lock vertex buffer channel with D3DLOCK_DISCARD.
- `FUN_00535910` — unlock vertex buffer.
