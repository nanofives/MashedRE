## Mechanical description
- One-arg state query (int audioSource). Returns uint (effectively bool).
- If `audioSource[+0x28] & 0x100` (cached "fresh" bit): returns `(audioSource[+0x28] >> 9) & 1` (the requested play-state).
- Else: queries driver-side state.
  - Hardware (`caps[+0x50] & 8 == 0`) AND attached buffer exists: calls FUN_005bc020(audioSource + 0xdc) → returns 1 if non-zero (i.e. "is playing").
  - Software AND attached buffer exists: reads software-voice state at `audioSource[+0x11c] + 0x28`. Returns 1 if state is 2 or 4 (playing/looping); else 0.
- Returns 0 if no buffer attached.
