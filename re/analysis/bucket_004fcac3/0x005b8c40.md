## Mechanical description
- One-arg audio-source update (float param_1 — actually `int audioSource` passed in EAX with float-prologue; signature mis-detected).
- Reads driver caps `iVar7 = *(int*)(audioSource + 0x94)` and software-mode flag `uVar10 = caps[+0x50] & 8`.
- Early return: if `audioSource[+0x84] == 0` (no attached buffer): zero `audioSource[+0x88]` and clear flag bits 0x05fc, return.
- If buffer flag bit 1 set AND driver `caps[+0x14] != 0`: process 3D position update.
  - Hardware (`uVar10 == 0`) path: reads pos at audioSource[+0xa0..0xa8], velocity at audioSource[+0xac..0xb4], min/max distance at +0xd4/+0x98. Calls FUN_005a9280 (3D-listener resolver) which writes local_4c (computed sample-rate scalar) + local_48.
    - If flag bit 0x1000 (paused/idle): force pos float assignment; else interpolate with 0.5 weight (`_DAT_005cc32c == 0.5`).
    - If `caps[+0x50] & 0x40` (frequency-modulation capability): updates frequency via DAT_007d4110-equivalent vtable+0x40 (Set-Frequency).
    - Else (no FM): set audio matrix on `audioSource[+0xe0]` vtable+0x30.
  - Software (`uVar10 != 0`) path: copies 3D fields directly into the software-voice struct at `audioSource[+0x11c] + 0x4c..+0x60` (negates y components consistent with hardware variant).
- Process other dirty flags:
  - bit 0x1000: lazily reads `buffer[+0x2c]` (sample-count?) into audioSource[+0x2c] as float.
  - "dirty-block" bit (any of 0x07 in caps[+0x50]) AND state bits 9 and 1 set: emit DSound-style message `&DAT_005e7100` (0x34 bytes from audioSource+0xe8) via vtable+0x10 of audioSource[+0xe4].
  - bit 0x20 (frequency dirty): hardware → vtable+0x44 (Set-Frequency, multiplied by audioSource[+0x3c]); software → write audioSource[+0x2c] to voice+0x34 and OR voice[+0xcc] |= 4.
  - bit 0x40 (pan dirty): hardware → FUN_005aeb90 to compute pan and pass via vtable+0x3c; software → copy audioSource[+0x30] to voice+0x3c and OR voice[+0xcc] |= 0x10.
  - bit 0x80 (volume dirty): hardware → vtable+0x40 (Set-Volume scaled to dB-100..0); software → write audioSource[+0x34] to voice+0x48 and OR voice[+0xcc] |= 2.
- Software-mode final: calls FUN_005c75b0 and writes result to audioSource[+0x88], clears dirty bits.
- Hardware-mode final: status poll via FUN_005bc020 on DSound buffer. If state bit 0x100 (need-restart): calls vtable+0x48 (Stop), then vtable+0x34 (SetCurrentPosition) with audioSource[+0x90], and if state bit 0x200 (need-Play) calls vtable+0x30 (Play, with loop flag from bit 0x800).
- If state bit 0x400 (position-rewind): calls vtable+0x34 with audioSource[+0x8c].
- Polls GetCurrentPosition via vtable+0x10 or fallback to direct audioSource[+0x90]. Stores at audioSource[+0x8c], computes `+0x88 = (raw / blockAlign) / channels` (sample-frame count) using buffer[+0x38] and buffer[+0x39] as block-align nibble and channels.
- Dirty-bit 0x10 (DSP/effect parameter dirty): if `audioSource[+0xe4]` (DSP slot) is non-NULL, emits one of four parameter blobs depending on caps[+0x50] bits 1/2/4: low/midi/reverb DSPs (uses _pow for log-scale via `pow(10, gain/dB-scale)`).
