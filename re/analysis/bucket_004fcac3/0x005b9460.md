## Mechanical description
- Two-arg buffer-attach (int audioSource, int newBuffer). The "core" setter for the audio source's attached DSound/SW buffer.
- If currently playing (FUN_005b9320 != 0): stop via FUN_005b92e0(audioSource, 0).
- If software-mode (`caps[+0x50] & 8`): calls FUN_005c75c0(audioSource[+0x11c], newBuffer[+0x4c] or 0) — wire software voice to the new sample.
- If hardware-mode AND old buffer existed: calls FUN_005bbf30(audioSource + 0xdc) — release old DSound buffer.
- Zeros `audioSource[+0x84]` (the old buffer pointer).
- Resets pitch/rate scalars: `audioSource[+0x38] = 1.0f` (0x3f800000), `audioSource[+0x3c] = 1.0f`.
- If `newBuffer != 0`:
  - Hardware-mode: calls FUN_005bbf60(caps[+0x54], newBuffer + 0x5c, audioSource + 0xdc) — clone the DSound buffer from a template. If clone returns 0, abort without setting +0x84 (failed-clone path leaves +0x84 = 0).
