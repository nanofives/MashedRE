## Mechanical description
- Four-arg sample-mix-with-saturation (int outPtr, int srcA, int srcB, uint byteCount).
- Treats outPtr/srcA/srcB as 16-bit PCM short* arrays. `byteCount` is in bytes; `param_4 >>= 1` produces a sample count.
- Main loop processes 4 samples per iteration:
  - For each of 4 16-bit lanes: `sum = srcA[i] + srcB[i]` as 32-bit int.
  - Saturate: if sum >= 0x8000 → clamp to 0x7fff; if sum < -0x7fff → clamp to -0x7fff; else passthrough.
  - Store the 16-bit truncation to outPtr[i].
- Tail loop handles the remaining `(N % 4)` samples one-at-a-time with the same saturation logic.
- Equivalent to a saturated 16-bit PCM mix-add (audio summing for 2 voices).
