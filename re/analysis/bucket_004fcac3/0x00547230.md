## Mechanical description
- Six-arg ray-triangle intersection (float *rayOrigin, float *rayDir, float *V0, float *V1, float *V2, float *outT). Returns 1 on hit, 0 on miss.
- Classic Möller-Trumbore algorithm:
  - `edge1 = V1 - V0`, `edge2 = V2 - V0`
  - `P = rayDir × edge2` (computed as `(d.y*e2.z - d.z*e2.y, d.z*e2.x - d.x*e2.z, d.x*e2.y - d.y*e2.x)`)
  - `det = P . edge1`
  - If `det <= _DAT_005e4574` (epsilon, likely 0.0001f or similar): return 0 (back-facing or parallel)
  - Compute `epsilon_band = det * _DAT_005cea1c` (epsilon constant) and band-clamp boundary `band = det - epsilon_band`.
  - `T = rayOrigin - V0`
  - `u = T . P`. If `u < epsilon_band || u > band`: miss.
  - `Q = T × edge1`
  - `v = Q . rayDir`. If `v < epsilon_band` or `(u + v) > band`: miss.
  - `t = Q . edge2`. If `t < epsilon_band` or `t > band`: miss.
  - Else `*outT = t / det`, return 1.
- The variable `*param_6 = fVar1` is also written on certain miss paths (stores the unscaled intersection parameter before scaling) — appears intentional for debugging or batched checks.
