## Mechanical description

3-arg **saturating int32 → int16 PCM pack** (the mixer output-conversion leaf).
- `param_1` = `int16*` destination, `param_2` = `int32*` source accumulator, `param_3` = count.
- Per source sample `s` (int32):
  - if `s < -0x800000` → output `0x8000` (int16 min, −32768),
  - else if `s < 0x7fff01` → output `(int16)(s >> 8)` (arithmetic shift: drops 8 fractional/headroom
    bits, converting a 24.8-style fixed value to 16-bit),
  - else → output `0x7fff` (int16 max, +32767).
- The loop is hand-unrolled: a main loop running `param_3 >> 1` iterations converts 4 samples each;
  a tail loop converts 2 samples per iteration for the remainder when `param_3` is odd. (Net: writes
  `4*(param_3>>1)` samples plus a 2-sample tail.)
- Returns void.

This is the final downmix saturating conversion (matches the "16-bit PCM saturated mix" backend note).
