# FUN_0040e180 — collision-pair finder (max-distance pair among live cars)

## Mechanical description

Two `int*` out-parameters. Iterates the per-car table `PTR_PTR_005f2770+0x34..0x44`
(four entries, stride 4 = cars 0..3) in a *nested* loop. For each (outer, inner)
pair where:
- `*(int *)(PTR_PTR_005f2770 + iVar * 4 + 0x34) != 0` (slot occupied), and
- `FUN_0046c7b0(idx) == 1` (car alive predicate), and

it fetches the car's transform via `FUN_0046d4a0(&local_34, idx)`, reads world
coordinates from offsets +0x30/+0x34/+0x38, computes the delta vector
(`local_18`/`14`/`10`), and calls `FUN_004c3ac0(&local_18)` (Vec3Magnitude, RVA
0x004c3ac0 — landed hook). The maximum magnitude across all valid pairs wins;
its (outer, inner) indices are stored in `local_3c` (outer) and `iVar3` (inner).

After the double loop, three trailing reconciliations resolve the case where
one index stayed at -1 (no pair found in a row but a pair found earlier). The
fallbacks write through `*param_1` and `*param_2`.
