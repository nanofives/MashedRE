## Mechanical description

- Signature (inferred): `int FUN_00550980(?, void* buf, int element_size, int count, StreamCtx* ctx)`
  - param2 = buf, param3 = element_size, param4 = count, param5 = stream context ptr
- Computes `total_bytes = element_size * count` (IMUL EDX, [ESP+10])

- Returns `result_bytes / element_size` (DIV ESI) — mirrors `fread` element count semantics

## ABI (confirmed via disassembly, Ghidra pool10, 2026-05-22)

Original is plain `__cdecl` with 4 stack args in left-to-right push order:

| Param | Register/Offset (inside, after PUSH ESI) | Role |
|-------|------------------------------------------|------|
| param_1 | [ESP+0x8] | buf |
| param_2 | [ESP+0xC] | element_size |
| param_3 | [ESP+0x10] | count |
| param_4 | [ESP+0x14] | ctx |

ECX is **not** the calling convention. `MOV ECX, [ESP+0x14]` at 0x00550987 loads ctx from the stack into ECX for use in the body — NOT because this is thiscall.

Dispatch: `fn = *(void**)(*(void**)(ctx+0x38) + 0x28 + 0x0C)` — single deref from `[EAX+0xC]` where `EAX = *(ctx+0x38) + 0x28`. **Not** a double-deref vtable lookup.

Bug fixed 2026-05-22: original reimpl had `(ctx, buf, element_size, count)` (wrong order) and double-deref dispatch. Corrected to `(buf, element_size, count, ctx)` with single-deref dispatch.
