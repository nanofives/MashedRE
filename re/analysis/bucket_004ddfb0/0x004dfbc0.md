## Mechanical description
- Signature `void FUN_004dfbc0(int dst_raster, int src_image)`.
- 8bpp palette-raster conversion entry. Allocates 256-byte palette `abStack_4108` and 16388-byte scratch image `local_4008` on stack (note `__chkstk` injection).
- Recurses into itself when source image needs an intermediate 32bpp re-render (see end branch).
- Branches on `*(byte *)(src+0x23) & 0xf` (palette/format code) == 4 (palette-indexed):
  - D3DFORMAT 0x1b: builds 256-entry palette table by per-pixel formula `(byte)(pbVar[1] & 0xe7 | pbVar[2] >> 3) >> 3 | *pbVar & 0xe0` — R3G3B2 palette pack. Then maps source pixels through palette.
  - D3DFORMAT 0x34 (D3DFMT_A4L4): builds 256-entry table by `lum = (b[1]*0x3b + b[2]*0xb + b[0]*0x1e)/100 >> 4 | (b[3] & 0xf0)`.
  - D3DFORMAT 0x1c: builds 256-entry alpha-only table (`b[3]`).
  - Other 4-bit format: builds 256-entry NTSC-luma-with-alpha-premult table.
- For non-palette source (depth 4/8 means already palette-indexed): pixel-copies and palette-copies straight through.
- Otherwise (32bpp source): allocates new image via FUN_004cdca0(w, h, 0x20), FUN_004cdd60 to init, FUN_004daa40 (raster lock), FUN_004d9050 (color-fill 0x3f800000=1.0f), FUN_004d9530 (palette generation), FUN_004da4f0 (palette-from-image solver), FUN_004daaf0 (raster unlock), then optionally detects format via FUN_004dee40 and recurses into self with the intermediate.
