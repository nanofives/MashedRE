## Mechanical description

Loads per-glyph metrics by scanning the font atlas bitmap directly (METRICS3 variant).
Supports multi-atlas sources and UTF-8 multi-byte codepoint decoding. Retries up to 4
rounds (outer `local_54` counter).
Signature (from decomp): `undefined4 *FUN_00555ff0(undefined4 *param_1, undefined4 param_2, undefined4 param_3, byte *param_4)`.
- `param_1`: font context pointer.
- `param_2`: font name / lookup key.
- `param_3`: base path arg passed to FUN_00550a20.
- `param_4`: line-read buffer.

Outer retry loop (up to 4 rounds, `local_54` = 0..3):

**Setup (per round):**
1. Saves RenderWare heap state via `FUN_004c5ca0()` / `FUN_004c5c80(DAT_00912a20)` (0x00556002).
2. Reads `param_1[0x4c]` (glyph capacity, offset 0x130); defaults to 0x100 (0x00556018).
3. Calls `FUN_005c4c60` / `FUN_005c4d30` to resize and get raw glyph buffer pointer
   `local_1c` (0x00556027).
4. Zeroes `capacity * 0x20` bytes from `local_1c` (0x00556030).

**Header parse:**
5. Reads a line with `FUN_00550a20(param_4, 0x100, param_3)` (0x00556049).
6. Scans via vtable+0x104 with format `"%s %s %s"` for texture name into `&DAT_0091320c`
   and key into `param_2` (0x00556054). If scan returns 0, or not 1 or 2, jumps to cleanup.
7. Calls `FUN_004ce2d0(DAT_00912a0c)` to activate VFS path (0x0055607a).
8. Opens texture handle `piVar8` via `FUN_004c5cb0`: round 1 uses NULL path (0x00556082);
   round 2+ uses `&DAT_0091320c` (0x0055608b). If NULL, sets return NULL (0x0055609c).
9. Stores `piVar8` at `param_1[local_54 + 5]` (0x005560a2) — supports up to 4 atlases.
10. Reads atlas pixel dimensions: `iVar2 = *piVar8[0xc]`, `iVar3 = *piVar8[0x10]` (0x005560a8).
11. Atlas image handle `local_68`:
    - If first atlas seen: creates via `FUN_004cdca0(iVar2, iVar3, 0x20)` then locks via
      `FUN_004cdd60(local_68)` (0x005560b7).
    - If already exists but dimensions differ: unlocks old via `FUN_004cdd00(local_68)`,
      creates new (0x005560c2).
12. Calls `FUN_004d52d0(local_68, *piVar8)` to copy/blit atlas pixels into the image
    (0x005560d7).
13. Computes UV normalisation: `fVar5 = 1.0f / iVar2`, `fVar6 = 1.0f / iVar3` (0x005560e0).
14. Reads row stride `iStack_8` via vtable+0x104 with `&DAT_00623ee8` format (0x005560e9).

**Atlas scan loop (Y rows × X columns):**
15. Outer loop over rows `iStack_70 = 0..iVar3-1` (0x005560f5).
16. Inner loop over columns `iStack_5c = 1..iVar2-1` (0x00556106):
    - Reads pixel row pointer `pcVar4` from `local_68 + 0x14` (0x00556102).
    - Computes stride `iVar12 = *(local_68 + 0xc) >> 3` (bytes per row) (0x00556108).
    - Scans for glyph boundaries: compares run of `iVar12` bytes of row 0 vs row `iStack_70`
      for equality (`bVar23` loop) (0x00556119). If rows match (separator row), advances
      `iStack_70`.
    - For each column boundary detected: decodes the current `pbVar16` byte as UTF-8:
      - 1-byte: direct codepoint (0x00556460).
      - 2-byte `0xC0..0xDF`: `uVar10 = (b0 & 0x1f)<<6 | (b1 & 0x3f)` (0x005563a7).
      - 3-byte `0xE0..0xEF`: 3-byte decode (0x005563bc).
      - 4-byte `0xF0..0xF7`: 4-byte decode (0x005563d1).
      - 5-byte `0xF8..0xFB`: 5-byte decode (0x005563e8).
      - 6-byte `0xFC..0xFD`: 6-byte decode (0x005563ff).
      - If `uVar10 >= 0x10000`: sets error flag `local_54 = 4`, return NULL (0x00556700).
    - Codepoint routing:
      - If `uVar10 < 0x80`: writes `(short)local_50` at `param_1 + uVar10*2 + 0x24` (0x005566bf).
      - Else: `param_1[4] |= 2`, stores `uVar10` in extended table (0x005566c9).
    - Writes 0x20-byte glyph entry at `local_1c[iStack_3c]`:
      - `[+0x00]` = `((iStack_5c - iVar13) - 3) / ((iStack_70 + iVar11) - iStack_70 - 3)` (0x00556703)
        [UNCERTAIN U-5654 — denominator recalculates `iVar11 - 3` as height; -3 suggests
        border/shadow pixel exclusion]
      - `[+0x04]` = piVar8 (texture pointer) (0x005566e5)
      - `[+0x08]` = `(iStack_18 + 0.5f) * fVar5` (U left) using `_DAT_005cd088` (0x005566e9)
      - `[+0x0c]` = `(iStack_70 + 0.5f) * fVar6` (V top) (0x005566f0)
      - `[+0x10]` = `(iStack_5c - 0.5f) * fVar5` (U right) using `_DAT_005cc32c` (0x005566f7)
      - `[+0x14]` = `((iStack_70 + iVar11) - 0.5f) * fVar6` (V bottom) (0x005566fd)
      - `[+0x1c]` = `(byte)local_54` (round index / source atlas index) (0x005566e3)
    - Grows glyph buffer by 0x20 if capacity reached (same realloc pattern as 0x00555af0).
    - Increments `local_50` (glyph count).
    - Handles line-buffer refill if `pbVar16` exceeds 0x7f bytes past `param_4` start
      (0x00556349).

**Round completion:**
17. After scan: sets `param_1[1] = (float)iVar11 - 0.5f` (line-height), 
    `param_1[2] = iStack_8 / line_height` (aspect ratio), clears `param_1[3]` (0x00556710).
18. Increments `local_54`; if < 4 loops back to read the next atlas header line.

**Teardown:**
19. Unlocks atlas image: `FUN_004cdd00(local_68)` if non-zero (0x0055672a).
20. If return pointer still valid: stores glyph count, calls `FontCtx_BuildExtTable`,
    resets ext-table sentinels (0x0055672f).
21. Restores RenderWare heap: `FUN_004c5c80(local_4)` (0x00556772).
22. Returns font context pointer or NULL.

Body: 0x00555ff0..0x0055677b (2188 bytes).
