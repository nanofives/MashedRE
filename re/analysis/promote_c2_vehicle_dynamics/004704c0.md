## Purpose

Vehicle entity init / spawn. For slot `param_1` (0–15): clears two collision tables,
zeros per-slot physics state fields, OR-in flags 0x20003, writes three 1.0f groups,
builds spawn matrix from params, resets VehicleWheelContactSolver, sets RW scene node
transform, marks slot active, then triggers a full physics state reset.

## Shape

**Signature:**
```c
int FUN_004704c0(uint param_1, undefined4 param_2, float param_3,
                 undefined4 param_4, undefined4 param_5, undefined4 param_6);
// param_1 = slot index 0..15
// param_2 = spawn position / handle
// param_3 = height offset
// param_4 = orientation
// param_5 = physics material index
// param_6 = secondary param to FUN_0046d400
```
**Returns:** 1 on success, 0 if slot >= 16

**Writes (key addresses):**

- `DAT_0088178c + slot*0xD04` (4×0xC4): fill 0xFFFFFFFF (collision table B)
- `DAT_00881ed4 + slot*0xD04` |= 0x20003 (physics flags)
- `DAT_00881f14 + slot*0xD04` |= 0x20003
- `DAT_00881ec8[slot]`, `DAT_00881f08[slot]` → 1.0f groups
