## Mechanical description

- Signature `RwTexture* f(RwStream* stream)`.
- Calls `FUN_004cc5e0(stream, 1, &chunk_size, &chunk_type)` —
  `RwStreamFindChunk(stream, type=1, ...)`.
- Validates `chunk_type` ∈ `[0x35000 .. 0x37002]` (same range as
  0x004cf2b0). Else RwError `0x80000004`.
- Reads 4-byte packed flags into `local_110`. If `(local_110 & 0xf000) == 0`,
  back-fills with `(local_110 & 0xf00) << 4` (legacy compatibility:
  promotes pre-version-10 addressing mode encoding to current).
- Saves current global filter/address state via `FUN_004c5820()` /
  `FUN_004c5850()` (getters from the 0x004c5800 cluster — already
  plated; sister `Set` helpers are also in the same band).
- If lower-nibble (`local_110 & 0xff`) ∈ {3, 4, 5, 6}: applies
  flag-write `FUN_004c5800(1)` and `FUN_004c5830((local_110 & 0x10000) == 0)`.
  Else `FUN_004c5800(0)` / `FUN_004c5830(0)`. These set the
  "active sampler addressing-mode" global default.
- Reads name (128 bytes `local_80`) and mask-name (128 bytes `local_100`)
  strings via `FUN_004d8810` (string-stream-read).
- Calls `FUN_004c5cb0(local_80, local_100)` — texture lookup/load
  by name+mask (from the existing 31 plates; this is in the 0x004c5xxx
  cluster covered by README as "RW resource registry dispatch").
- If lookup hits: restore saved filter/address state; if texture
  refcount field at `+0x54 == 1` (newly-created), write the lower
  16 bits of `local_110` into `+0x50` (filter/address mode field).
  Call plugin-registry chunk read (`FUN_004e1b60`) for plugin data.
- If lookup misses: skip plugin read (`FUN_004e1df0` — "skip plugin
  chunks") and return 0.
- Returns the loaded texture or 0.
