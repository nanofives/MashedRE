## Mechanical description

- Signature `int f(int frustum_ctx, int bbox_min_ptr)`.
- Iterates 6 planes at `frustum_ctx + 0x94` (stride 5 floats, same as
  0x004cbb70). Each plane has a `{nx, ny, nz, d}` head plus a 4-byte
  trailer holding **three bytes of axis-sign indices** at offsets
  +0x10..+0x12 of the plane struct.
- Per plane: each sign-byte selects which corner of the AABB to test:
  - Byte at `+0x10` (= `pfVar1[4]` lower-byte) -> X picker
  - Byte at `+0x11` (offset 0x11) -> Y picker
  - Byte at `+0x12` (offset 0x12) -> Z picker
- Each byte is used as `(byte) * -0xc + base_offset` — i.e. picks
  either `bbox_min[i]` (byte=0) or `bbox_max[i]` (byte=1) via the
  fixed AABB layout `bbox_min` at +0x0c, `bbox_max` at +0x18 (deltas
  of -0xc match this).
- Computes the dot product against the picked corner, compares
  against plane d. If `dot < d`, AABB is fully outside that plane —
  return 0.
- Returns 1 if AABB is at least partially inside all 6 planes.
- Mechanical shape: **RwCameraFrustumTestBBox** (P/N-vertex AABB cull).
