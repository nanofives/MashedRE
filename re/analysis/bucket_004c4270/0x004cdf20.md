## Mechanical description

- Signature `RwImage* f(RwImage* dst, RwImage* src)`.
- **Pre-check**: `dst->width == src->width && dst->height == src->height`,
  otherwise raises RwError `0x8000000a` ("dimension mismatch").
- **If `dst->depth` is palettized (4 or 8)** and source isn't:
  - Constructs a temporary 32bpp scratch image via the same
    RwImageCreate path as 0x004cdca0 (allocates header, sets
    width/height/depth, links into global list).
  - Calls `FUN_004cdd60` (RwImageAllocatePixels — [STUB]) on temp.
  - Calls either `FUN_004ceaf0` (same-depth-copy) or `FUN_004ceba0`
    (cross-depth converter) on the temp.
  - Cleans up the temp via the same freelist-and-list-remove dance
    as RwImageDestroy.
  - Promotes `dst->depth` to 0x20 internally, allocates new dst pixels.
  - Copies between the temp and dst when their depths match;
    otherwise calls cross-depth converter `FUN_004ceba0(dst, temp)`.
- **If `dst->depth == 0x20`** (true-color path): direct path.
- **Else** (unsupported depth combo): RwError `0x80000009`.
- **Pixel copy** (palettized destination):
  - For each row of `dst->height`, for each col of `dst->width`,
    read 1 byte from `src->pixels + col * src->stride_or_4`, look up
    palette `src->palette[byte*4 + 3]` (the A channel of that palette
    entry), store into `dst->pixels[3]`. Walk both with appropriate
    strides.
  - 32bpp destination: byte-by-byte copy A channel only (channel [3]).
- Returns dst.
