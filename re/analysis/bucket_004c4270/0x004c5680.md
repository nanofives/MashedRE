## Mechanical description

- Signature `float* f(float* mat, float* axis_out)` (16-float RwMatrix
  in, 3-float out).
- Picks the largest diagonal element of `mat[0], mat[5], mat[10]`.
- For the dominant diagonal, computes
  `local_X = 2 * (mat[dom] + 1.0)`. Constant `_DAT_005cc320` again
  carries `1.0f`.
- Off-diagonal slots: `local_Y = mat[off1+offN] + mat[offN+off1]`,
  `local_Z = mat[off2+offN] + mat[offN+off2]` (symmetric combinations
  of the off-diagonal entries).
- Calls `FUN_004c3b90` (FastInvSqrt) on
  `local_X^2 + local_Y^2 + local_Z^2` to normalize.
- Writes `axis_out[0..2] = (local_Y, local_Z, local_X) * invLen`
  (note the permutation — Ghidra's path-order shows ordering swaps per
  branch).
- This is the canonical rotation-axis extraction for the "trace near
  zero or negative" branch of Matrix-to-AxisAngle conversion. Pairs
  with caller 0x004c5470 (`RwMatrixQueryRotate`-shape).
