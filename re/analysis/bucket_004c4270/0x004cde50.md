## Mechanical description

- Signature `RwImage* f(RwImage* image)`.
- Branches by `image->depth` (slot [3]):
  - **Depth 4 or 8** (palettized): iterates over the palette at
    `image->palette` (slot [6]). Palette size = `1 << depth` entries
    (16 or 256). For each 4-byte palette entry `[R, G, B, A]`,
    computes `max(R, G, B)` into bytes [0..2] then stores into A
    (byte [3]). Mechanical: alpha = max RGB component.
  - **Depth 0x20 (32)** (true-color): iterates over the 2D pixel
    array — `width` columns by `height` rows, stride = `image->stride`
    (slot [4]). For each 4-byte pixel `[R, G, B, A]`, same
    `A = max(R, G, B)` operation. Note iterates only over `width`
    in the inner loop but reads `image->width` each iteration —
    classic `for (i = 0; i < image->width; ++i)` shape.
- Other depths: no-op, returns image unchanged.
- Mechanical shape: **RwImageMakeMask** — populates alpha channel
  from luminance-like max(R,G,B). Standard RW image utility.
