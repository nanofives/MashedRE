## Mechanical description

- Signature `int f(int viewport_ctx, HWND hwnd)`.
- Guards: requires `DAT_00912100 != 0` (engine initialized) and the
  request is for a different HWND than `DAT_007d4104` (current
  primary window).
- Calls `GetWindowRect(hwnd, &local_68)` (Win32 API).
- Destroys any prior rasters at `viewport_ctx + 0x60` (front buffer)
  and `+0x64` (back buffer) via `FUN_004c7650` (RwRasterDestroy).
- Copies a 14-DWORD presentation parameter block from
  `&DAT_009120e0` (a D3DPRESENT_PARAMETERS template) into stack.
  Updates `Windowed = (right == left)` and `BackBufferCount =
  (bottom == top)` (these are recycle-flag uses — the actual
  D3DPRESENT_PARAMETERS fields are width/height/depth being filled
  later).
- Calls IDirect3DDevice9 vtable +0x34 — **CreateAdditionalSwapChain**
  (vtable offset 0x34 in d3d9.h). Returns `iVar2 == 0` on success.
- On success: queries the new swap chain's backbuffer via vtable +0x14
  / +0x30 (GetBackBuffer / GetDesc on IDirect3DSwapChain9).
- Creates two new rasters via FUN_004c77c0 (RwRasterCreate) with
  format hints `0x82` (front) and `0x01` (back) — these are RW raster
  type flags.
- Writes the new rasters to `viewport_ctx + 0x60` / `+0x64` with
  metadata (HWND, byte flags from `DAT_00911b00 + ...`).
- Mechanical shape: **creates a secondary D3D9 swap chain** bound to
  a new HWND and wraps it as a pair of RwRasters. Used for split-screen
  or floating-window viewports.
