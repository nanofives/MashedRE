## Mechanical description
- Large per-projectile-collision handler. Reads `in_EAX` as a projectile slot index. Uses two parallel arrays:
  - `&DAT_007ea5e4` (int, slot-indexed) — owner / target reference.
  - `&DAT_007e9de4` (int, slot-indexed) — projectile "type" enum (switched 3..8).
  - `&DAT_007ec9e4` (float[16], slot-indexed by *0x10) — projectile state floats (pos at +0, +4, +8; another vec3 at +0x20..0x2c; sized).
  - `&DAT_007ea9e4` (int, slot-indexed by *8) — projectile color/RGBA-ish int constants (compared against `0xffXXYYZZ` patterns).
- Switch on type:
  - **case 3** (color `0xff968080` etc.): jittered direction emit + secondary particle if magnitude exceeds gate. Calls `FUN_0048aa50` (particle emitter with multi-args including 0x14 ID and float configs).
  - **case 4**: pick one of two source-positions randomly, normalize direction, emit `FUN_0048aa50` (ID 0x14), call `FUN_004c3ac0` (Vec3Magnitude) twice, store global `_DAT_007f0f94 / _DAT_007f0f98`. Conditionally scale + emit.
  - **case 5**: if track-piece index > 2 (matches "FUN_0046cc10 returns 3+"), compute pos delta to camera (FUN_00467210), broadcast effect with 360°, dispatch FUN_0041ede0.
  - **case 6**: branch on owner-int constants 0xff80ffff (-0xffff01) or 0xffff80ff (-0xff80). First: delta-to-camera + broadcast + FUN_004215c0 twice (50.0f arg). Second: configure + FUN_0048b230 + FUN_0041eda0 + FUN_0045ba00.
  - **case 7**: no-op.
  - **case 8**: query iVar2 from owner-int. If 0 AND timer > _DAT_005ccd10: `FUN_004922e0(slot, 1, 2, 0xff)` — particle/score popup with alpha 255. If 3 AND same: emit `FUN_0048aa50` with ID 10 (different from above).
- Uses many magic colors as ENUM-like identifiers in param_4 (RGBA-packed):
  - 0xff968080 (= "medium-red"), 0xffc88080, 0xffb48080, 0xff80c800, 0xff0064b4, 0xff0080c8, 0xff646464 (gray), 0xffa08080, 0xffff32ff, 0xffc88080, 0xffc8c8c8, 0xffdc8080, 0xffff8040, 0xffffffff.
  - Each maps to a different effect ID (1, 6, 7, 8, 9, 10, 13, 14, 15, 16).
