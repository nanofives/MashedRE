## Mechanical description

Serializes a string/blob pool into a caller-supplied dword buffer. Companion to 0x00510f66 (size query) and 0x00510e2f (insert).

Args: `param_1`=pool struct, `param_2`=output buffer (dword*), `param_3`=size or 0xffffffff.

Flow:
1. If `param_3 == 0xffffffff`, set required `uVar4` from 0x00510f66's formula. Else use `param_3` directly and reject if smaller than required → return 0x80004005.
2. Cap at 0x8000 (32K) — return 0x80004005 if exceeded.
3. Write header:
   - `out[0] = ((uVar4 - 1) & 0x7fff) << 0x10 | 0xfffe` — packed (size_quads-1) in high 16 bits, magic 0xfffe in low 16 bits.
   - `out[1] = pool[0]` — pool backing-context.
4. Walk linked list at `pool[+8]` via slot [4]:
   - If bucket-flag bit 4 is **clear** (bit means "skip alignment"): pad with 0xab bytes to next-4 boundary.
   - Copy `bucket[+4]` bytes from `bucket[+0]` (data) into output (dword loop + tail-byte loop).
   - Advance both write-ptr and cum-byte counter.
5. Pad final tail with 0xab bytes to fill the requested size.
6. Returns 0.

The 0xab fill is the **MSVC debug "fresh memory" marker** — useful for diagnostics but also valid in release because the consumer reads only the declared content lengths.
