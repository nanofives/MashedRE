## Mechanical description

String/blob pool intern/insert with optional dedup. Signature:
`HRESULT this->intern(char* data, char* len, uint flags, uint32_t* out_key)`

Args: `param_1` = pool struct; `param_2` = data ptr; `param_3` = length (or 0xffffffff for "compute as strlen"); `param_4` = flags; `param_5` = out-key.

Flow:
1. If `data == NULL` and `len != NULL`, return `0x80004005` (E_FAIL).
2. If `len == 0xffffffff`: compute as `strlen(data)` (byte-counting loop).
3. If `(flags & 1) == 0` (not "external-owned"), force `flags |= 2` ("auto-add"). [If both unset, dedup disabled.]
4. **Dedup pass** (if `flags & 2`):
   - Walk linked list at `[param_1 + 8]` via slot [4] (next-ptr).
   - For each bucket: if `bucket[+8] & 2` (dedup-flag set) AND `bucket[+4] == len` (length match): memcmp data — on full match, write `*out_key = bucket[+0xc]`, optionally `_free(data)` if `flags & 9 == 9`, return 0.
5. **Insert**: `operator_new(0x14)` — a 5-dword bucket. On null → return `0x8007000e` (E_OUTOFMEMORY).
6. If `flags & 1` (external): bucket[0] = data. Else: allocate fresh and dword/byte-copy.
7. If `(flags & 4) == 0`: align `pool[+4]` (total-bytes counter) up to next 4-byte boundary.
8. Set bucket fields: [0]=data, [4]=flags, [8]=pool[+4] (assigned key = current offset), [0xc]=current key, [0x10]=0 (next=NULL).
9. Update pool counter `pool[+4] += len`, append bucket via `**(pool[+0xc]) = bucket; pool[+0xc] = &bucket[+4]` (tail update).
10. If `out_key`, write key. Return 0.

This is a classic **interning string pool** / atom table: data+length keyed by ordinal id, with dedup-by-content option, external-or-copied data, and aligned cumulative-offset key generation.
