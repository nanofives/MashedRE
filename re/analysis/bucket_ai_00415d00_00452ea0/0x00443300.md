## Exact evaluation order (x87, from 0x00443383..0x00443432 listing)
The original keeps t, t², t³ and every coefficient in 80-bit ST registers, rounding
to float32 ONLY at the two final FSTP stores. Per component (X: [reg]; Z: [reg+4]):
- Q (quadratic coeff) = ((2*P0 - 5*P1) + 4*P2) - P3
- C (cubic coeff)     = ((3*P1 - P0) - 3*P2) + P3
- t³ precomputed once = t²*t (single 80-bit value)
- out = 0.5 * ( 2*P1 + ( (C*t³ + Q*t²) + (P2-P0)*t ) )
This 80-bit intermediate chain is NOT expressible in portable C (float locals are
32-bit) → a C transcription diverges 1-4 ULP (x87-association residual). The .asi
port therefore installs a naked-asm verbatim body for bit-identity.
