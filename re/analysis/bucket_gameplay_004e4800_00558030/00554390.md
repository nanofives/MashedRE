## Mechanical description

`int * FUN_00554390(int param_1)` — top-level binary vector-font loader from stream
`param_1`. Single caller FUN_00427880 (application page, font registration). This is the
function whose behavior the standalone B19 faithful-font work reimplemented for
FGDC20.RWF.

- `FUN_005551d0()` allocates the empty font object; abort → NULL.
- Reads 4-byte version word `local_2c` via FUN_004cc790; requires `local_2c >> 0x18 != 0`
  (nonzero high byte), else NULL.
- Reads 7 more u32 fields into font[0..4], font[0x49], font[0x4c]; then font[0x4a]
  (extended-char count).
- If font[0x4a] > 0: allocates `2*count` bytes via engine-global slot +0x108 with hint
  0x30190, zero-fills, stores at font[0x4b], then stream-reads `2*count` bytes into it
  (i16 extended-codepoint table). Alloc failure → `FUN_00555830(font)` + NULL.
- Reads 0x100 bytes into font+0x24 (256-entry ASCII glyph-index table).
- `FUN_005c4c60(font[0x4d], font[0x4c], 0x30190)` grows the glyph-record handle to
  glyph_count capacity; `FUN_005c4d30(font[0x4d])` resolves it; zero-fills
  `glyph_count << 5` bytes (0x20-byte records).
- Branch `*font == 0` (form 0, stroke/palette font):
  - `uStack_28 = FUN_004c5ca0()`; `FUN_004c5c80(DAT_00912a20)` (context push/scope set).
  - Per-glyph loop (font[0x4c] iterations, record cursor +0x20): reads 0x10 bytes to
    rec+0x08, 4 bytes to rec+0x00, 1 byte to rec+0x1c; zeroes rec+0x18.
  - Reads u32 `param_1` (palette count), then per entry: reads 0x20 bytes to stack,
    `FUN_004c5cb0(buf, 0)` creates a child object stored at font[5+i]; failure →
    destructor+NULL.
  - Second pass over glyph records: resolves `rec+0x04 = font[(char)rec[0x18..] index + 5]`
    (decompiled `piVar2[(char)piVar6[6] + 5]`, i.e. byte at rec+0x18 of the record block
    indexes the palette array); a NULL resolved entry aborts.
  - font[0x4f]=0, font[0x50]=0, font[0x4e]=FUN_00554940 (form-0 dispatch fn).
  - `FUN_004c5c80(uStack_28)` restores scope; returns font.
- Branch `*font != 0` (form !=0, path font):
  - Per-glyph loop: reads 4 bytes to local, 4 bytes to rec+0x00, 4 bytes to local (type);
    if type == 5: `FUN_004cc5e0(stream, 0x1a1, 0, 0)` then `FUN_00554200(stream)` loads a
    path sub-list, plus one more 4-byte read; stores the path at rec+0x18; one final
    4-byte read per glyph.
  - font[0x4e] = LAB_005547f0 (form-!=0 dispatch). Returns font.
- All mid-stream failures route through `FUN_00555830(font)` → NULL.
