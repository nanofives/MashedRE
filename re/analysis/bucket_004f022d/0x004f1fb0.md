## Mechanical description

- Signature: `u32 * FUN_004f1fb0(int *param_1, u32 *param_2, uint param_3)`.
- Triangle list → triangle strip converter (NvTriStrip-style; same shape as RenderWare strip-generation).
- Pseudo-pipeline:
  1. Allocate scratch array of `*(int*)(p1+4) << 2` bytes (triangle-ptr array).
  2. Fill scratch with pointers to triangles (stride 0x14 from `(p1+8)`).
  3. Sort via `FUN_005c24e0(piVar3, count, 4, &LAB_004f2420)` (qsort) using material-id comparator at +8 of each triangle.
  4. Count distinct materials (iVar12) — runs of identical `*(triangle+8)`.
  5. Allocate material-group descriptor array of `iVar12*0xc` bytes via `+0x108`.
  6. For each material group, sub-allocate triangles (stride 0x14), call `FUN_004f2570` (core strip-generator), then `FUN_004f3440` (strip-merger).
  7. For each strip in result, allocate strip-descriptor (12+strip.count*2 bytes), populate, store into output index array, free working buffers.
  8. Build final mesh header via `FUN_004f0970(iVar14)` (which dispatches through RwEngine vtable).
- Returns the resulting mesh-header object.
