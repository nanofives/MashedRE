## Analysis
Sweep of the Lua interned string table (`stringtable` / `strt` in Lua 5.x).

### lua_State field deductions
- `L+0x2c` = `strt.size` (hash bucket count)
- `L+0x30` = `strt.nuse` (live string count)
- `L+0x34` = `strt.hash` (pointer to bucket array)
- `L+0x60` = `totalbytes`

### TString node layout (offsets in bytes)
- `+0x08` = string length (`str_len`, used in `totalbytes` adjustment)
- `+0x0c` = `next` (linked list pointer in hash bucket)
- `+0x10` = GC mark (int; 0 = white/collectible, ≥2 = gray/black)

### GC two-white pattern
- Mark < 2 AND param_2==0 (not forced): clear mark to 0 (white-flip for next cycle), keep alive.
- Mark == 0 OR param_2==1 (forced): unlink and free.

`-0x15 = -21` = sizeof TString common header overhead.

Pattern: `sweepstrings` in `lgc.c` (Lua 5.x), called during GC sweep phase.
