# 0x004ddcb0 — FUN_004ddcb0 (D3D9 texture dimension clamp + POT round-up)

**RVA:** 0x004ddcb0  
**Size:** 0x004ddcb0..0x004ddd5f (0xb0 = 176 bytes)  
**Subsystem:** render  
**Confidence:** C2  
**Batch:** batch-render-7-s2  
**U-id:** U-5527

## Signature

```c
void FUN_004ddcb0(int *param_1, int *param_2, uint param_3);
```

- `param_1` — in/out: texture width.
- `param_2` — in/out: texture height.
- `param_3` — flags: bits 0..2 control non-POT path; bit 8 controls forced-square path.

## Mechanical description

1. Clamps width to `DAT_00911ff8` (MaxTextureWidth cap) and height to `DAT_00911ffc` (MaxTextureHeight cap).

2. Checks `_DAT_00911fdc & 2` (D3DPTEXTURECAPS_POW2):
   - If set (and `_DAT_00911fdc & 0x100` is clear or path requires POT): rounds both width and height up to next power-of-two using a bit-scan loop (counts leading bits, shifts 1 up).
   - Path depends on `param_3`: bits 0..2 == 0 or `param_3 >> 8` >= 0 for the inner branch.

3. Checks `_DAT_00911fdc & 0x20` (D3DPTEXTURECAPS_SQUAREONLY):
   - If set: forces `*param_1 = *param_2 = min(*param_1, *param_2)`.
