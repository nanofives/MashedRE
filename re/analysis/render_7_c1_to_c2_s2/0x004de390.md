## Mechanical description

First checks `*(byte*)((int)param_2+0x23) & 0xf` (the raster sub-format nibble). If non-zero: dispatches on `param_2[5]` (bits-per-pixel): 4, 8, 0x10, 0x18, or 0x20.

**4bpp/8bpp path (case 4, case 8):**
- `param_2[2] == 0` (no palette): expands byte to BGRA by replicating the grey value: `0xff000000 | (v << 16) | (v << 8) | v`.
- `param_2[2] != 0` (palette): looks up 32bpp entry from palette at `param_2[2]`.

**16bpp path (case 0x10):** switches on sub-format nibble:
- 1 (A1R5G5B5): expands to BGRA8.
- 2 (R5G6B5): expands to BGRA8 (A=0xFF).
- 3 (A4R4G4B4): expands to BGRA8.
- 10 (X1R5G5B5): expands to BGRA8 (A=0xFF).

**24bpp/32bpp path (case 0x18, 0x20):** switches on `*(int*)(D3D9base + obj + 0x18)` (D3DFORMAT enum):
- 0x16 (D3DFMT_X8R8G8B8): copies 3 bytes RGB, sets A=0xFF.
- 0x15 (D3DFMT_A8R8G8B8): copies all 4 bytes.
- 0x23 (D3DFMT_A2R10G10B10): 10-bit → 8-bit shift per channel.

**Fallthrough** (no matching sub-format): calls `FUN_004de800(param_1, param_2)` — the generic format-dispatch converter.
