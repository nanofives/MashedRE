## Mechanical description

Per-frame proximity scanner. Iterates the active vehicle set (up to 4, indexed by `DAT_007f1a14`) to detect when vehicles are close enough to interact (drive alongside, collide head-on, etc.) and writes events to the ring buffer consumed by `FUN_00422ba0`.

**Interaction table layout** (per-vehicle-pair):
- Base: `DAT_008995c4` (stride 0x4e per vehicle, second vehicle adds offset within row)
- `piVar8[0xc]` = "currently in contact" flag (1-bit)
- `piVar8[1]` = contact duration accumulator (delta-time based)
- `piVar8[-0x3f]` = score awarded for this pair interaction
- `piVar8[-0x3e]` = interaction category (1 = alongside, 2 = head-on)
- `piVar8[-0x3d]` = interaction timer cap (3000)

**Event types written to queue:**
| Type code | Meaning |
|-----------|---------|
| 0x3ff03 | "Driving alongside" event |
| 0x3ff04 | "Head-on" event |
| 0x3ff01 | "Near-miss" event [U-3592] |
| 0x3ff02 | "Proximity" event [U-3592] |
| 5 | Interaction-end event |

The threshold `0x2329` (= 9001 ms) is the contact-duration gate for scoring.
