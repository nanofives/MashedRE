## Mechanical description

Half-edge adjacency builder for a triangle mesh. `in_EAX` = triangle count. Allocates a scratch
table of `tri_count*0x48` bytes (3 half-edges per triangle, 0x18 bytes each), fills each
half-edge with its two vertex indices + owning-triangle index + (-1) twin, matches opposite
half-edges via `FUN_0047bc90`, counts matched pairs (`local_c`), allocates the compact output
array `local_c*0x18` into `*param_1` (count → `param_1[1]`), copies matched half-edges in, and
frees the scratch.

## Shape

(register arg), `param_1` = output {ptr, count}, `param_2` = triangle vertex-index array
(3 ushorts per tri, stride 8 shorts advanced as `param_2 + 4` dwords).

**Body (0x0047bcc0..0x0047be71):**
- `iVar7 = in_EAX*3` (half-edge count); alloc `in_EAX*0x48` scratch via vtable +0x108, zero it.
- Per triangle: fill 3 half-edge records (0x12 dwords = 0x48 bytes / 3) with vert pairs
  `(v0,v1),(v1,v2),(v2,v0)`, owning-tri index, twin=-1.
- Pair-match loop: for each unmatched half-edge call `FUN_0047bc90()` (returns EDX:EAX);
  on match set twin links and clear the matched slot.
- Count surviving matched edges → `local_c`; alloc `local_c*0x18` output → `*param_1`,
  `param_1[1]=local_c`; copy 6-dword (0x18-byte) records in.
- free scratch via vtable +0x10c.
