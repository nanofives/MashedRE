## Mechanical description
- Calls FUN_00496400 (0x00496400) with string "Operating System:\n" — printf-style logger.
- Zero-fills a local OSVERSIONINFOA struct (0x27 DWORD iterations).
- Sets _Stack_100.dwOSVersionInfoSize = 0x9c (156); calls GetVersionExA(&_Stack_100). On failure, retries with size 0x94 (148).
- Switch on dwPlatformId and version fields; writes integer code to DAT_00773920 (0x00773920) and logs OS name string:
  - dwPlatformId=0: DAT_00773920=7, "Microsoft Win32s"
  - dwPlatformId=1, MajorVer=4, MinorVer=0: DAT_00773920=2, "Microsoft Windows 95" (+ "OSR2" if szCSDVersion[1]=='C')
  - dwPlatformId=1, MajorVer=4, MinorVer=10 (0xa): DAT_00773920=3, "Microsoft Windows 98" (+ SE suffix if szCSDVersion[1]=='A')
  - dwPlatformId=1, MajorVer=4, MinorVer=0x5a (90): DAT_00773920=4, "Microsoft Windows Me"
  - dwPlatformId=2, MajorVer<5: DAT_00773920=1, "Microsoft Windows NT"
  - dwPlatformId=2, MajorVer=5, MinorVer=0: DAT_00773920=5, "Microsoft Windows 2000"
  - dwPlatformId=2, MajorVer=5, MinorVer=1: DAT_00773920=6, "Whistler" (Windows XP pre-release name)
  - Unknown platform: "Unknown OS"
- For NT platform (dwPlatformId=2) with old-form OSVERSIONINFO (BVar1==0): queries registry key "SYSTEM\CurrentControlSet\Control\ProductOptions", "ProductType" to distinguish Workstation/Server variants; calls FUN_00496400 with product strings.
- For NT platform (dwPlatformId=2) with extended OSVERSIONINFOEX (BVar1!=0): reads wSuiteMask and wProductType fields from uStack_68 to determine Professional/Personal/Server/DataCenter/Advanced variants.
- Logs version string "version %d.%d %s (Build %d)\n" or "%s (Build %d)\n" depending on MajorVer.
- Returns void.
- Cross-reference: DAT_00773920 set here is the switch discriminant in FUN_00402750 (cases 1,5,6 → pcVar4=0; cases 2,3,4,7 → pcVar4=1), controlling PIZ filesystem mode selection.
