## Addendum 2026-05-21 — FP-associativity discovered

While verifying via draw_quad_observe, a reimpl bug surfaced that's worth recording: the orig computes `(short_screen_dim * param) * scale_const` strictly left-to-right. Writing `param *= short_screen_dim * scale_const` produces a different bit-pattern under IEEE-754 (FP multiplication is non-associative). The reimpl uses explicit `x = sW * x * scale_x` to preserve order. This is a general lesson for any RW Im2D function that scales coordinates against `short` screen dims.
