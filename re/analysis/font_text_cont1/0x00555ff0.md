## Body summary (0x00555ff0‚Äì0x0055677b)

Populates `font_ctx` with glyph metrics by scanning bitmap pixels directly from an atlas texture. Called from LAB_00555910 when the METRICS3 section tag is found. More complex than the Met parser ‚Äî extracts glyph bounding boxes by comparing row equality in the bitmap.

### Setup (0x00555ff0‚Äì0x00556090)
1. Save/set TXD (same pattern as FUN_00555af0): `FUN_004c5ca0()` / `FUN_004c5c80(DAT_00912a20)`.
2. Initialize glyph data buffer (same resize+zero pattern).
3. Retry counter `local_54 = 0`.

### Outer retry loop `LAB_00556093` (up to 4 rounds)
Each round processes one page (one atlas texture).

1. `FUN_00550a20(line_buf, 0x100, filename_base)` ‚Äî read line
2. `vtable+0x104(line_buf, "%s %s\n", tex_name, &DAT_0091320c)` ‚Äî sscanf
3. Look up texture: `FUN_004c5cb0` ‚Üí `piVar8` (tex dict entry)
4. Create bitmap scan context: `local_68 = FUN_004cdca0(tex_width, tex_height, 0x20)` ‚Äî allocate image (S-3691)
5. `FUN_004cdd60(local_68)` ‚Äî allocate pixel data (S-3692, RwImageAllocatePixels C1)
6. `FUN_004d52d0(local_68, *piVar8)` ‚Äî bind texture pixels to image (S-3693)
7. `fVar5 = 1.0f / tex_width`, `fVar6 = 1.0f / tex_height` ‚Äî UV scale factors
8. `vtable+0x104(line_buf, "%d\n", &iStack_8)` ‚Äî read char height override

### Glyph extraction loop
Scans bitmap rows: compares each row to first row via `iVar12`-byte block compare. When a row differs from row 0, it marks a glyph strip boundary.

Per-glyph, once strip is located:
- Reads character code from `line_buf`: handles UTF-8 multi-byte sequences (2‚Äì6 byte encodings, codepoints up to 0xFFFF)
- If char_code > 0xFFFF: `local_54 = 4` (error) ‚Üí `local_64 = NULL`
- Glyph entry write (stride 0x20, same layout as FUN_00555af0):
  - `[+0x00]` = `(strip_w - 3) / (strip_h - 3)` ‚Äî advance ratio (subpixel-trimmed)
  - `[+0x04]` = `piVar8` (texture ptr)
  - `[+0x08]` = `(x_start + 0.5f) / tex_w` ‚Äî u_left (DAT_005cd088 = 0.5f offset)
  - `[+0x0c]` = `(row_start + 0.5f) / tex_h` ‚Äî v_top
  - `[+0x10]` = `(x_end - 0.5f) / tex_w` ‚Äî u_right (DAT_005cc32c = -0.5f offset applied)
  - `[+0x14]` = `(row_end - 0.5f) / tex_h` ‚Äî v_bottom
  - `[+0x1c]` = `local_54` (page index byte)
- ASCII/extended char table update: same as FUN_00555af0

### Finalization (same pattern as FUN_00555af0)
`FUN_004cdd00(local_68)` ‚Äî destroy scan image (S-3694); `FUN_00555f20(font_ctx)` ‚Äî build ext table; write `font_ctx[1/2/3]`.
