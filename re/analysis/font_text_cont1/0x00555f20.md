## Body summary (0x00555f20–0x00555feb)

Post-metrics-load finalization: builds the compact extended-character lookup table from the temporary global array `DAT_0091325c`.

Called at the end of both FontCtx_LoadMetrics_Met (0x00555af0) and FontCtx_LoadMetrics_Atlas (0x00555ff0).

### Procedure

1. `count = *(int*)(font_ctx + 0x130)` — extended glyph count (number of non-ASCII glyphs loaded)
2. If `count <= 0`: return immediately (no extended chars).
3. Scan `DAT_0091325c[0..count-1]` for non-zero entries:
   - Track min char code (`iVar8`, init = 0xFFFF)
   - Track max char code (`iVar5`, init = 0)
4. Allocate short array: `(max - min + 1) × 2 bytes` via `vtable+0x108`.
5. Fill array with `0xFFFF` (sentinel = "not mapped").
6. Second pass: for each `iVar7 = DAT_0091325c[i]` where `iVar7 > 0`: `table[iVar7 - min] = (short)i` (glyph_index).
7. Write to font_ctx:
   - `*(uint*)(font_ctx + 0x128)` = table size (max - min + 1)
   - `*(int*)(font_ctx + 0x124)` = min char code (base)
   - `*(undefined4**)(font_ctx + 0x12c)` = table ptr
8. Returns `font_ctx`.
