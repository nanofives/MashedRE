## Analysis

Called directly from the concrete `RwCameraBeginUpdate` implementation at `0x004c2bbc` (`CALL 0x004d80d0`), before the render dispatch via `*(rwGlobals+0x4c)`.

Walks the dirty-frame linked list stored in `rwGlobals` and propagates local-to-world matrices down each dirty frame hierarchy.

### Dirty-list structure

Sentinel head at `DAT_007d3ff8 + 0xbc` (`0x007d3ff8+0xbc` = rwGlobals dirty list). Each dirty entry is a pointer to `frame + 8` (the `inDirtyList.next` link within the frame). The loop variable `puVar3 = frame+8`; frame base is thus `puVar3 - 8`.

- `puVar3 + -5` = `frame + 3` = `RwObject.privateFlags`
- `puVar3[0x22]` = `*(frame + 8 + 0x88)` = `*(frame + 0x90)` = objectList sentinel head
- `puVar3[0x24]` = `*(frame + 8 + 0x90)` = `*(frame + 0x98)` = firstChild

### privateFlags bit semantics

| Bit | Mask | Meaning |
|-----|------|---------|
| 0 | 0x01 | Frame has a parent hierarchy (LTM must be concatenated) |
| 2 | 0x04 | This frame's modelling matrix is dirty (must recompute LTM) |
| 3 | 0x08 | Child hierarchy needs traversal (set by `FUN_004c0e50`) |

### Two paths

**Path A — bit 0 clear (root frame with no parent):**
- Calls sync callbacks on attached objects (via `obj+0x10` function ptr called with the object).
- Iterates first-level children via `frame+0x98`, then siblings via `frame+0x9c`.
- For each child: calls sync callbacks, clears bit 3 (`&= 0xf7`), recurses deeper via `FUN_004d8300`.
- No matrix multiply (root LTM = modelling matrix; no parent to concatenate with).

**Path B — bit 0 set (non-root frame):**
- If bit 2 set: copies 64 bytes (16 dwords) from `frame+0x10` (modelling) to `frame+0x50` (LTM candidate — intermediate step before child concat; or direct LTM if no parent).
- Calls sync callbacks on objects.
- For each child: if child or parent has bit 2 set, calls `FUN_004c4600(child+0x50, child+0x10, *(child+4)+0x50)` — this is `child->ltm = child->modelling × parent->ltm`.
- Clears bits 2,3 from child (`&= 0xf3`), recurses via `FUN_004d8280`.
