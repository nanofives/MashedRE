## Mechanical description

- Signature: `void FUN_005ae920(uint *param_1, uint param_2)`
- param_1 = pool header pointer.
- param_2 = address of the item being freed (as uint).
- Pool header layout:
  - param_1[0] = element size (bytes)
  - param_1[1] = elements per block
  - param_1[2] = bitmap dword count (uVar2)
  - param_1[4] = linked list head of allocated blocks
  - param_1[6] = flags

**Steps:**
1. Walk block list: iterate blocks from param_1[4] (the block head) forward via `*(block)[0]` until param_1+4 (sentinel).
2. For each block: compute block_start = `block + uVar2 + 8`. Check if `param_2` falls within `[block_start, block_start + param_1[1] * param_1[0])`.
3. When found:
   - Compute item index: `uVar4 = (param_2 - block_start) / param_1[0]`.
   - Clear the allocation bit: byte `block + 8 + (uVar4>>3)`, bit `7 - (uVar4 & 7)` (MSB-first bitmap).
4. Unlink block from current list.
5. If param_1[6] bit 1 set (pool compaction flag): count remaining set bits in bitmap.
   - If any bits still set: re-link block at the free-list head.
   - If all bits clear: `FUN_004522d0(block)` — release the now-empty block back to the heap.
6. Else (no compaction): re-link block at free-list head unconditionally.

## Role

Fixed-size pool allocator "free" function. Searches all blocks for the item's block, clears the
bitmap bit to mark it free, then optionally releases empty blocks to the heap (when the
pool-compaction flag is set). Used by all audio object types that use pool allocation.
