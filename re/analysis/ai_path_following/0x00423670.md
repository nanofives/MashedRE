## Analysis

Runtime spline editor. Processes gamepad/keyboard input to edit AI spline data in-game. Uses the same `DAT_007f1a54` sub-mode and `DAT_007f1a68`/`DAT_007f1a6c` selectors as `FUN_00444ff0`.

### Input dispatch (`DAT_007f1a54` mode × `DAT_007f1a64` key)

**Mode 0 (overview/line select):**
- Key 0 → enter spline-edit (race line), `DAT_007f1a54=6`, `DAT_007f1a68=0`
- Key 1 → enter spline-edit (current line), `DAT_007f1a68=iVar2`
- Key 2 → enter spline-edit (slow lines), `DAT_007f1a68=2`
- Key 3 → enter spline-edit (current line), `DAT_007f1a68=iVar2`
- Key 5 → enter save-mode, `DAT_007f1a54=3`
- Key 7 → enter clear-data mode, `DAT_007f1a54=1`
- Key 8 → enter race-options, `DAT_007f1a54=4`, `DAT_007f1a74=0`

**Mode 1 (clear-data confirm):**
- Key 2 → cancel (return to mode 0)
- Any other → call `FUN_00423630()` (execute clear)

**Mode 3 (save-data confirm):**
- Key 2 → cancel
- Any other → call `FUN_00423540()` then return to mode 0

**Mode 4 (race options):**
- Key 4 → toggle `DAT_007f1a74 ^= 1`

**Mode 6 (spline point edit):**
- Key 2 → `*(float*)(spline + DAT_007f1a98 * 8) = _DAT_007f1a58 + _DAT_005cc320`; `*(float*)(spline + DAT_007f1a98 * 8 + 4) = _DAT_007f1a5c - _DAT_005cc320` — **set point XZ to vehicle 0 position**; if at count boundary: increment count.
- Key 3 → **insert point** (shift array right from `spline + count*8 - 4` down to `DAT_007f1a98`); place new point; increment count. Max 0x40 points.
- Key 5 → exit edit mode (return to mode 0)

**Point navigation (mode 6, via key inputs):**
- `DAT_007f107{2,3}` (up/down): increment/decrement `DAT_007f1a98` (edit point), clamped to [0, count].
- `DAT_007f107{5,4}` (left/right): increment/decrement `DAT_007f1a6c` (spline index 0..2).

**Vehicle position tracking:**

`FUN_0046d4a0(&local_10, 0)` → vehicle-0 position pointer:
- `_DAT_007f1a58 = *(float*)(local_10 + 0x30)` (X position)
- `_DAT_007f1a5c = *(float*)(local_10 + 0x38)` (Z position)

These are the values placed into spline points at Key 2 (mode 6) — the editor snaps a point to the current vehicle-0 position.

### Save path (mode 3 → FUN_00423540)

Calls `FUN_00423480()` then writes AI data to file via `FUN_004cc230(2,2,...)` + `FUN_004cc580`.

### Live-recording path (`DAT_007f1a64 == 5`)

When mode-6 is active with key 5: monitors `DAT_007f1043` (recording flag?) and `DAT_00644100` (frame counter):
- State 0 → state 1 on first trigger; sets `DAT_007f1a70 = 1`
- State 1 → state 2 after 0x1e (30) frames; sets `DAT_007f1a70 = iVar2`
- State 2 → after 3 more frames: `DAT_007f1a70 = 1`

## Structural finding: spline edit format

Point insertion (mode 6, key 3) reveals the in-memory spline array format directly:
- Each point = 2 consecutive `float`s (XZ)
- Stored at `base + index * 8`
- Count at `base + 0x200` (= `*(int*)(base + 0x200)`)
- Maximum 0x40 = 64 points per spline

This is the same layout documented in `FUN_00443300`.
