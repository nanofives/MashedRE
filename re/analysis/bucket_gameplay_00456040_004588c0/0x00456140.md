## Mechanical description

`param_2` (32-bit RGBA). Stack locals `local_14`/`local_10` receive a copy of `param_2`
(used as the per-axis SIZE), while `param_2` itself is overwritten with `0xffffffff`
(white) and used as the COLOUR — i.e. the sprite is drawn white-tinted at a size
derived from the passed colour dword.

1. `local_14 = local_10 = param_2` — stash original arg as size.
2. `param_2 = 0xffffffff` — force white.
3. `FUN_004769f0(&param_2)` — set sprite colour (white).
4. `FUN_004769d0(&local_14)` — set sprite size (from original param_2).
5. `FUN_00474de0(param_1, local_c, 0x3dcccccd /*0.1f*/)` — build a transform matrix
   into `local_c[12]` at position `param_1`, scale 0.1.
6. `FUN_004769a0(local_c)` — bind that matrix as the sprite transform.
7. `fVar1 = FUN_00472650(0, 0x43b40000 /*360.0f*/)` — random float in [0, 360).
8. `FUN_00476a30((float)fVar1)` — set sprite rotation to that random angle.
9. `FUN_00476d00(&DAT_006887d0)` — draw/queue into the shared sprite list `0x006887d0`
