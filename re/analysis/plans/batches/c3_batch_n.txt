# Per-function workflow (drift-staged variant)
1. Read re/analysis/<sub>/<rva>.md AND the existing reimpl file end-to-end.
2. Verify the impl matches the analysis note's signature exactly.
3. If `arg_type` in the analysis note is in `re/frida/diff_template.js`'s
   supported set: add a `hooks_registry.py` entry with the right signature
   + test vectors (>=10).
4. If `arg_type` is NOT supported: STOP-AND-ASK; queue a harness-extension
   line in PROMOTION_QUEUE.md rather than authoring a new arg_type here.
5. Run `py -3.12 re/frida/run_diff.py <hook_name>`. Expect GREEN.
6. If RED: debug ONE hook at a time. Don't promote the others first; the
   issue may be cross-cutting.
