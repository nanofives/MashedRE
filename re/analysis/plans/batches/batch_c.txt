Session 13 — track_world_initial_sweep  (FIRST-PASS, no seed RVAs)
================================================================================

Pool slot:   Mashed_pool12 (pre-assigned)
Model:       Sonnet 4.6 — hard cap 20 RVAs
Skill used:  parallel-fanout first-pass
Bucket dir:  re/analysis/track_world_initial_sweep/

# Mission
The track subsystem currently has ONE row in hooks.csv. This is the most
alarming coverage gap in the tracker. Identify the track-loading entry
points by string xrefs (TRACK_*, .piz paths under TOASTART/TRACKS/,
checkpoint / lap detection strings), document them, and walk depth-1 into a
C1 map of at least 20 functions.

# Workflow
1. Startup with $SLOT=Mashed_pool12.
2. Entry-point discovery:
   a. mcp__ghidra__search_defined_strings for "TRACKS\\", ".piz", "ARCTIC",
      "CITY", "EGYPT", "checkpoint", "lap", "course", "track" (case-insensitive
      where possible).
   b. For each hit, get xrefs_to. The functions referencing these strings
      are candidate track entry points.
   c. Cross-reference with prior_art/MashedFileExtractor to see what track
      file format expectations exist.
3. Pick the top 3 entry points (the ones with the most string xrefs and
   that look like a track-loading orchestrator, not just a string-print).
4. function_callees depth-1 from each entry point. Aggregate.
5. Cap subset at 20. Overflow → DEFERRED as track_world_initial_sweep-cont1.
6. For each RVA: function_at + decomp + per-RVA C1 plate.
7. Update re/analysis/subsystem_map.md with the discovered track entry
   points (the "Entry points: TBD" line under `### track`).
8. File stubs/uncertainties/deferrals.
9. Shutdown + scribe-queue append.

# Output
- ≤20 per-RVA .md files in re/analysis/track_world_initial_sweep/
- ≤20 C1 hooks.csv rows (with `subsystem=track`)
- Update to re/analysis/subsystem_map.md: track entry points filled in
- A REPORT.md summarizing the entry-point search and what was found
- Queued row in SCRIBE_QUEUE.md

# STOP-AND-ASK if
- No track strings are present in MASHED.exe (would mean track names live
  in .piz / .rws assets, not in the binary — entry-point strategy fails).
- The track callee tree explodes past 60 (signal: track is wired through
  many small leaf functions — confirm the slicing strategy).

   "raycast" string constants.

4. If nothing surfaces from strings, the collision data may not be
   string-tagged. Try: get the xrefs FROM vehicle-physics functions
   (sessions 15 & 16) that look like they query the world. The shared
   callees are the collision-query layer.

5. Startup with $SLOT=Mashed_pool13.
6. Build subset from the candidates. Cap 20. Overflow → cont1.
7. Per-RVA C1 plates. Identify the BSP / mesh struct if possible.
8. Draft re/analysis/structs/track_collision.md (the struct format).

# Output
- ≤20 per-RVA .md files
- ≤20 C1 hooks.csv rows (subsystem=track or physics depending on what you find)
- Possibly re/analysis/structs/track_collision.md
- Queued row in SCRIBE_QUEUE.md

  collision pipeline may be in a separate module — file as DEFERRED with
