Session 91 — breadth_unmapped_0040x  (FIRST-PASS, 20 RVAs)
================================================================================

Pool slot:   Mashed_pool0 (pre-assigned)
Model:       Sonnet 4.6 (claude-sonnet-4-6) — hard cap 24 RVAs (this session: 20)
Skill used:  parallel-fanout first-pass discovery
Bucket dir:  re/analysis/breadth_unmapped_0040x/

# Mission
Drain 20 unmapped Ghidra functions in the 0x00400000–0x0040ffff range.
After 12 batches of cont-drain, the 0x0040xxxx range still has unmapped
function objects (compiler-generated thunks, cold demo branches, etc.).
Identify the mappable ones and write C1 plates; flag the rest as
`[UNREACHABLE]` or `[COMPILER-GENERATED]` in DEFERRED.

# Inputs
At session start, run via the Ghidra MCP:
```
mcp__ghidra__function_list (paginated; filter address range 0x00400000-0x0040ffff)
```
Then:
```bash
# Filter for RVAs not yet in hooks.csv
awk -F, '$1' hooks.csv | sort > /tmp/hooks_rvas.txt
# diff against function_list output (one RVA per line)
comm -23 <(sort /tmp/funclist.txt) /tmp/hooks_rvas.txt | head -30
```

# Workflow
1. Acquire $SLOT=Mashed_pool0 (fall back).
2. For each of the top 20 unmapped RVAs:
   a. Decompile via Ghidra MCP.
   b. If the function is <50 bytes and has no callers: flag
      `[UNREACHABLE]` and file in DEFERRED with re-pickup condition
      "after dead-code analysis."
   c. If the function is a compiler-generated thunk (FidDB matches a
      CRT/RW library function): cite the FidDB match, write a short
      C1 plate, mark library-rename pending.
   d. If the function is mappable: standard C1 plate with `## Mechanical
      description` + `## Constants` + `## Uncertainties` + `## Stubs
      encountered`.
3. After 20 done:
   - Append Queued row to SCRIBE_QUEUE.md.
   - One commit at session end.

# STOP-AND-ASK if
- The 20 unmapped slots are entirely [UNREACHABLE] / compiler-generated
  — surface; the slice may be exhausted of meaningful work.
- A function matches a Ghidra-built-in library function whose name
  conflicts with an existing FidDB match — do NOT rename this session;
  flag for the sweep.
