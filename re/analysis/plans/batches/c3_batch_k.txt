# Per-function workflow (repeat for each RVA, then commit ONCE at end)
1. Read the analysis note end-to-end (path from the candidates table).
2. Confirm all `callees_depth1` are at C2+ in hooks.csv (filter v4 already
   validated this; double-check at session start — drift can creep in).
3. Confirm the signature has a matching arg_type in
   `re/frida/diff_template.js`. If not: STOP and queue a note in
   `re/PROMOTION_QUEUE.md` asking for a harness extension; DO NOT invent.
4. Author the reimpl in
   `mashedmod/src/mashed_re/Frontend/MenuCluster_k1.cpp`:
     - One C++ function per RVA, all in the same file.
     - Cite the RVA + a few disasm bytes in the comment header.
     - `extern "C" __declspec(dllexport) <ret> __cdecl <Name>(<args>) { ... }`
     - `RH_ScopedInstall(<Name>, 0x00<rva>);`
5. Wire build:
     - Append `"%SRC%\Frontend\MenuCluster_k1.cpp" ^` to mashedmod\build.bat
       (one append for the whole file, not per RVA).
     - Append a hooks_registry.py entry per RVA: arg_type matches diff_template.js;
       >=10 test vectors covering 0/MAX/sign-bit/alternating bits.
6. Build:
     `cmd /c mashedmod\build.bat > log\build_batch-k-s1.txt 2>&1`
     Halt on failure.
