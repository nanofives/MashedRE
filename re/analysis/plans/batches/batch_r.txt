================================================================================
BATCH R — promote-c2 wave 4 (RenderWare engine init + .piz mount + video
            + launch handshake — Phase-3 engine boot path)
================================================================================

SIXTEEN parallel sessions. Pool slots pre-assigned Mashed_pool0..Mashed_pool15
(one per session, no race on `acquire`).

PREREQUISITE: batch_q `ghidra-sweep` complete. Sessions cross-reference
batch_q boot-chain C2 plates (display init, WinMain init).

GOAL — UNLOCK PHASE 3 ENGINE BOOT PATH FOR THE STANDALONE EXE

The standalone target needs reimplemented RenderWare init, .piz mount,
D3D9 device creation, and the launch.exe handshake. Today the boot
subsystem is at 0 C3 — the chain is entirely C1/C2. This batch closes the
Phase-3 gap from the roadmap that's been mostly skipped: every global
accessed during boot needs a documented type, and `WinMain` must be C3+.

Three clusters, all heavy on Ghidra (no Frida, no build):
  A.  RenderWare engine init + driver-system dispatcher        sessions 113..118
  B.  .piz mount + .txd / .rws asset loaders                   sessions 119..124
  C.  Video / D3D9 / window creation + launch handshake        sessions 125..128

A C1→C2 promotion is mechanical: read the existing C1 plate + decomp,
assemble a `## Why C2` block citing constants/offsets at RVA, list
explicit [UNCERTAIN] tags with U-IDs, file new stubs for any unmapped
callees, and flip hooks.csv `confidence` C1→C2. Same per-session shape
as batch_q.

All sessions follow re/SESSION_RULES.md § "Parallel-fanout scribe-queue
pattern". After all 16 finish committing, run `ghidra-sweep`.
