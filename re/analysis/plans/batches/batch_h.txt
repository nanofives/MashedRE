Session 48 — struct_extract_phase5  (DOCUMENTATION ONLY)
================================================================================

Pool slot:   Mashed_pool5 (pre-assigned)
Model:       Sonnet 4.6 — no RVA cap (documentation, not analysis)
Skill used:  none (pure markdown authoring + Ghidra read)
Bucket dir:  re/analysis/structs/

# Mission
Phase 5 subsystems (frontend, hud, save, input, audio, track, physics,
vehicle, ai, render) each need their shared structs documented per
S-DoD requirement #3. Several drafts exist already (hud_state,
gamesave_layout, vehicle_dynamics, vehicle_damage, gamepad_state,
particle_emitter, bsp_struct_a, path-following state at DAT_0089a4cc).
This session consolidates and fills in the MISSING structs.

# Prerequisites
This session MUST run after Sessions 43, 44, 47 have committed their
per-RVA .md files. Sessions 43 (render_frame depth-3) and 47 (TXD
texture) produce critical offsets this session consumes.

# Target structs (in priority order)
1. **race_state** — referenced by race_state subsystem (7 C1 rows) and
   leaderboard_d3 (30+ rows). Find the global pointer/root via grep
   across the leaderboard + race_state plates.
2. **camera_state** — camera_follow has 10+ unmapped plates but no
   struct doc. Look at re/analysis/camera_follow/ for shared offsets.
3. **dispatch_table_sfx** — audio_sfx_dispatch_d3 wrote a dispatch
   table doc per Batch B; consolidate it into structs/.
4. **rw_engine_globals** — the RW init code reads from a small set of
   DAT_ globals (DAT_007739f0, DAT_00773414, etc.). Document the layout
   inferred from sessions 25, 26, 39.
5. **input_dinput_state** — gamepad_state.md was drafted in
   sweep-20260511; check whether it's complete or needs another pass.

# Inputs
- ALL per-RVA .md files in re/analysis/*. Grep them for DAT_ references.
- Existing struct files in re/analysis/structs/ (do not rewrite — only
  amend or add new files).
- ROADMAP.md for S-DoD requirement #3 wording.

# Workflow
1. Startup with $SLOT=Mashed_pool5.
2. For each target struct above:
   a. Grep ALL per-RVA .md files for the DAT_ root candidate.
   b. Cluster by offset, width, sign.
   c. Document offset, width, sign, first read RVA + cite, first write
      RVA + cite, tentative semantic name (if multiple callers agree),
      [UNCERTAIN U-NNNN] if disagreement.
   d. Write re/analysis/structs/<struct_name>.md.
3. If a draft already exists, AMEND it (do not overwrite).
4. NO hooks.csv mutations. NO scribe-queue row needed (no master writes).
5. Shutdown.

# Output
- Up to 5 new or amended struct files in re/analysis/structs/
- A REPORT.md at re/analysis/structs/REPORT_phase5.md summarizing:
  - struct → DAT_ root → field count → confidence level
  - which subsystems' S-DoD this unblocks
  - any U-IDs filed for unresolved fields

# STOP-AND-ASK if
- A struct has fewer than 3 cross-references in the per-RVA plates
  (signal: not enough evidence to commit a struct shape — file as
  DEFERRED with re-pickup condition "after subsystem reaches N C1 rows").
- The same DAT_ root is referenced as DIFFERENT struct shapes from
  different subsystems (union-shaped struct or aliased global — design
  decision needed; do not silently pick one shape).
