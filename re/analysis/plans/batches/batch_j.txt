BATCH J — DEFERRED cont-drain (mid-size buckets)
================================================================================

Six parallel sessions. Pool slots pre-assigned (skip locked 2/4 — fall back to
ghidra_pool.sh acquire if stale-locked).

PREREQUISITE: batch_i must be fully drained by ghidra-sweep before batch_j
starts. Sessions cross-reference batch_i plates that won't be in master Ghidra
until batch_i's sweep commits.

This batch targets the mid-size unresolved cont-drain buckets (10-16 rows each).
Each session pulls its own RVAs at start time via grep on DEFERRED.md, so the
candidate list reflects the actual state at execution time (not at batch-
generation time). Sessions drift-skip any RVAs already at C1+ in hooks.csv
and fill the 20-RVA budget by walking depth-1 callees.

All sessions follow re/SESSION_RULES.md § "Parallel-fanout scribe-queue pattern".

After all 6 sessions finish committing, run `ghidra-sweep` to drain into master.

Session 55 — rw_engine_init_d2_cont2  (FIRST-PASS, ~12+8 cont-drain)
================================================================================

Pool slot:   Mashed_pool0 (pre-assigned)
Model:       Sonnet 4.6 (claude-sonnet-4-6) — hard cap 24 RVAs (target: 20)
Skill used:  parallel-fanout first-pass
Bucket dir:  re/analysis/rw_engine_init_d2_cont2/
Drains to:   re/SCRIBE_QUEUE.md (Queued); ghidra-sweep picks up later

# Mission
DEFERRED.md has ~12 unresolved rows tagged `rw_engine_init_d2-cont2`. Drain
all of them + ~8 depth-1 callees discovered during work.

# Inputs
At session start, run:
```bash
grep -E "^\| D-[0-9]+ \|" DEFERRED.md | grep -v "RESOLVED" | grep -v '~~' \
    | awk -F'|' '$5 ~ /rw_engine_init_d2-cont2/ {print $2, $3}' | head -25
```
That produces the bucket's current unresolved RVAs. Drift-skip any already in
hooks.csv at C1+. Use the rest, walk depth-1 of any to fill the 20-budget.

# Workflow
1. Acquire $SLOT=Mashed_pool0 (fall back to `ghidra_pool.sh acquire` if locked).
2. Pull candidate RVAs via the grep above; cross-check against hooks.csv.
3. Per-RVA C1 plates in re/analysis/rw_engine_init_d2_cont2/<rva>.md per
   re/SESSION_RULES.md format.
4. Mark resolved D-rows as RESOLVED in DEFERRED.md.
5. Append Queued row to SCRIBE_QUEUE.md.
6. One commit at session end.

# Output
- ≤20 per-RVA .md files in re/analysis/rw_engine_init_d2_cont2/
- ≤20 new C1 hooks.csv rows (subsystem=render mostly)
- ~12 DEFERRED rows RESOLVED
- Queued row in SCRIBE_QUEUE.md

# STOP-AND-ASK if
- bucket grep returns < 8 unresolved rows (most already drained by batch_i
  spillover) — confirm with user before pivoting to depth-1 discovery only.
