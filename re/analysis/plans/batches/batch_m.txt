Session 73 — promote_c2_audio_dsound  (C1→C2, 20 RVAs)
================================================================================

Pool slot:   Mashed_pool0 (pre-assigned)
Model:       Sonnet 4.6 (claude-sonnet-4-6) — hard cap 24 RVAs (this session: 20)
Skill used:  parallel-fanout C1→C2 promotion
Bucket dir:  re/analysis/promote_c2_audio_dsound/
Drains to:   re/SCRIBE_QUEUE.md (Queued)

# Mission
Promote 20 C1 audio_dsound rows to C2. The audio subsystem has 202 C1 rows
across audio_dsound/audio_dsound_cont1/audio_dsound_d3/audio_dsound_d5/
audio_promote_c2_rws_loader/audio_rws_loader/audio_rws_loader_cont1/audio_music/
audio_music_cont1. This session targets the dsound device-layer cluster
(005a9xxx–005bcxxx range).

# Inputs
At session start, run:
```bash
awk -F, 'NR>1 && $4=="C1" && $3=="audio"' hooks.csv \
  | awk -F, '$1 ~ /^005a[a-f9]|^005ba|^005bb|^005bc/' \
  | head -30
```
Pick the first 20 with an existing C1 plate under
`re/analysis/audio_dsound*/` or `re/analysis/audio_rws_loader*/`. If a row
has no plate yet, SKIP and substitute the next one — promote-c2 demands a
plate to upgrade.

# Workflow
1. Acquire $SLOT=Mashed_pool0 (fall back to ghidra_pool acquire on stale lock).
2. For each of the 20 RVAs:
   a. Read the existing C1 .md plate and Ghidra decomp.
   b. Author `re/analysis/promote_c2_audio_dsound/0x<rva>.md` with
      frontmatter `confidence: C2` and three sections:
      - `## Signature` (canonical signature with arg types, calling conv,
        return-type semantics)
      - `## Why C2` (numbered evidence list: each row cites RVA + constant
        or offset; every magic number resolved; every indirect call's
        target or [UNCERTAIN U-XXXX])
      - `## Open uncertainties` (zero or more [UNCERTAIN U-XXXX] entries —
        file U-IDs starting from UNCERTAINTIES.md high-water mark)
   c. Flip hooks.csv row's confidence C1→C2 in place; do NOT add a second
      row. Append the new plate path to the `file` column (semicolon-
      separated if a C1 plate path is already there).
   d. For any unmapped callee referenced by the function, file an S-ID
      in STUBS.md (only if not already present).
3. After 20 done:
   - Append Queued row to re/SCRIBE_QUEUE.md.
   - One commit at session end (NOT per RVA).

# STOP-AND-ASK if
- A function dispatches through an IDirectSound* vtable that has not yet
  been seeded in Ghidra COM-import types. Mark [UNCERTAIN U-XXXX], do NOT
  guess the COM method. Re-pickup condition: after COM-imports session.
- A C1 row turns out to have a stale plate (decomp differs from .md) — do
  not silently rewrite; file U-ID and leave C1.
- More than 5 of the 20 candidates already have a C2 row from prior work
  (drift) — that signals the bucket is mostly drained; surface and ask.
