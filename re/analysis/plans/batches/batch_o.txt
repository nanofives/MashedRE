================================================================================
BATCH O — promote-c3 wave 1 (Frida-side C2→C3, parallel-fanout)
================================================================================

Six parallel sessions. Each session is a WORKTREE (not a Ghidra pool slot)
— Frida hooks build their own `mashed_re_dev.asi` and run a per-hook
A/B diff against MASHED.exe in isolation. The Frida pool is parallel-safe
(per memory: 3-up parallel hook diffs proven at 2.1 s wall time; MASHED.exe
has no single-instance mutex — extends to 6-up).

PREREQUISITE: batch_n fully drained by ghidra-sweep before batch_o starts.
The C2 backlog after M+N sweep is ~476 rows; this batch consumes 30 of
them across 6 sessions × 5 candidates each.

Each session's deliverable:
  1. 5 hook implementations under
     `mashedmod/src/mashed_re/<Subsystem>/<Class>.cpp`
  2. Each registered via `RH_ScopedInstall(Method, 0xRVA)` in `InjectHooks()`.
  3. 5 rows in `re/frida/hooks_registry.py` (RVA, export name, signature,
     arg types, test inputs).
  4. 5 path1 diffs via `py -3.12 re/frida/run_diff.py <hook>` — bit-identity
     A/B against the original.
  5. 5 path2 installer verifications via
     `py -3.12 re/frida/run_verify_hook.py <hook>` — confirm inline-JMP
     installs.
  6. hooks.csv `confidence` flipped C2→C3; `frida_diff` field populated
     with `log/diff_<hook>.csv`; `status` flipped to `mapped`.
  7. Queued row in `re/PROMOTION_QUEUE.md` per the format in that file.
  8. ONE commit at session end on branch `c3/batch-o-sN` (where N is the
     session number 85-90).

C3 candidate selection criteria (apply STRICTLY):
  - Pure: no side effects on globals; no writes to DAT_ outside [local
    scope]; no calls into the runtime that mutate observable state.
  - Leaf-y: callees must be C3+ already (FastSqrt, FastInvSqrt,
    Vec3Magnitude qualify as established C3) OR be inlined-trivial.
  - Deterministic given inputs: no time reads, no RNG, no thread-local
    indices that vary across runs.
  - Frida-hookable signature: __cdecl or __stdcall preferred; __thiscall
    OK with explicit EAX-this in registry; void(void) acceptable but
    test inputs reduce to "call once and observe globals" — narrow A/B
    surface so do this last.
  - C2 row has a complete `## Why C2` block (constants cited at RVA, no
    [UNCERTAIN] in the body of the function — uncertainties in the
    `## Open uncertainties` section are OK if the function still
    behaves deterministically).

REFUSE C3 promotion on a candidate where any of the criteria fails;
substitute the next candidate from the subsystem's C2 backlog.

After all 6 sessions commit, the user runs `frida-sweep` (or this opens
a foreground review) to merge the 6 worktree branches onto main, resolve
the predictable text-file conflicts (hooks.csv, hooks_registry.py,
re/PROMOTION_QUEUE.md), rebuild the canonical `.asi`, and run the
integration A/B against ALL 30 hooks at once.

Session 85 — c3_audio_leaves  (C2→C3, 5 candidates)
================================================================================

Worktree branch: c3/batch-o-s85
Pool slot:       Mashed_pool0 for Ghidra cross-ref reads only (no writes)
Model:           Sonnet 4.6 (claude-sonnet-4-6)
Skill used:      hook-author + diff-original + re-classify

# Mission
Promote 5 audio C2 functions to C3. Audio has cleanly-shaped leaves —
pure transforms with no global side effects — that A/B well.

# Candidates (pre-vetted, drift-skip+substitute if any has degraded)
| RVA        | Function           | Why C3-friendly |
|------------|--------------------|----------------|
| 0x005aea10 | aligned_alloc_wrap | pure: size+4, stores base at aligned-4; void*(size_t) |
| 0x005aea40 | thin_free_wrap     | pure trampoline to FUN_004522d0; void(void*) |
| 0x005aec00 | byte_reverse       | in-place buffer endian-swap; void(buf, len) |
| 0x005aee20 | bit_scan_forward   | pure 0..31 bit scan loop; always called with 0x28; int(uint32) |
| (substitute) | (audio C2 leaf from batch_m Session 73/77 output) | |

# Test vectors (for hooks_registry.py)
- aligned_alloc_wrap: call with (0, 1, 16, 1024, 4096); record returned ptr alignment + tag-4-bytes.
- thin_free_wrap: pair with allocs above; check no exception.
- byte_reverse: input buffers [0x01..0x10] of lengths {0, 1, 2, 4, 8, 16}.
- bit_scan_forward: inputs {0x28, 0x01, 0x80000000, 0xFFFFFFFF, 0x00010000}.

# Workflow
1. Open `c3/batch-o-s85` worktree.
2. For each of the 5 RVAs:
   a. Read the C2 plate; confirm pure/leaf/deterministic.
   b. Author the hook (gta-reversed style) under
      `mashedmod/src/mashed_re/Audio/AudioMemory.cpp` (or shared file).
   c. Add to `InjectHooks()` with `RH_ScopedInstall`.
   d. Append row to `re/frida/hooks_registry.py`.
3. Build via `mashedmod\build.bat`.
4. `py -3.12 re/frida/run_diff.py <hook>` per RVA — must show byte-identity.
5. `py -3.12 re/frida/run_verify_hook.py <hook>` per RVA — confirm install.
6. Flip hooks.csv C2→C3 (via re-classify skill). Populate frida_diff col.
7. Append Queued row to re/PROMOTION_QUEUE.md.
8. One commit at session end.

# STOP-AND-ASK if
- A path1 diff shows mismatched bytes — fix the implementation; do NOT
  promote on a failing diff (NO overclaiming per CLAUDE.md).
- A function's byte sequence in MASHED.exe differs from the version
  anchor — abort the session (binary drift) and surface.
- The build fails — fix and retry; do NOT skip.
