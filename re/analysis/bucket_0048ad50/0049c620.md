## Mechanical description
- Wait loop: spins while `*(int *)(param_1 + 0xb4) != 0`, draining the message
  queue with `PeekMessageA(&msg, NULL, 0, 0, 0)` + `Sleep(1)`.
- After loop, calls `GetQueueStatus(QS_POSTMESSAGE = 8)`. If
  `((status >> 16) & 8) != 0` (a posted message present), wakes up the thread by
  posting an empty `PostThreadMessageA(GetCurrentThreadId(), 0, 0, 0)`.
- Param_1 is via ECX (`__fastcall`); offset +0xb4 is a "busy" or "in-progress" flag.
