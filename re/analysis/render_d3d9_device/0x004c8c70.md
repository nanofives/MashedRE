## What it does

Full D3D9 device shutdown. Corresponds to `_rwDeviceSystemFn` case `rwDEVICESYSTEM_STOP` or `rwDEVICESYSTEM_CLOSE`. Mirrors `criterion-rwg37/core/driver/d3d9/d3d9device.c` stop/close sequences.

**Sequence:**
1. Clears `DAT_007d4120` (IDirect3DDevice9* field — see note below)
2. Releases `DAT_007d4130` (depth-stencil or back-buffer surface)
3. Calls `FUN_005cb42b` (unknown; deferred)
4. Clears MXCSR FTZ bit if `DAT_007d458c` set (SSE float mode cleanup)
5. Releases shader constant pool at `DAT_007d4568` (mirror of FUN_004c8690's setup)
6. Releases `DAT_007d4570` (index or vertex buffer pool)
7. Unbinds all D3D states on device `DAT_007d4110`: 8 texture stages, indices, 4 stream sources, pixel shader, vertex decl, vertex shader
8. Releases texture object array at `DAT_007d414c` (COM Release + alloc free for each)
9. Calls cleanup routines `FUN_004cfe40`, `FUN_004dcdc0`, `FUN_004db550`
10. Calls `IDirect3DDevice9::Release()` via `+0x08` on `DAT_007d4110` → destroys device
11. `DAT_007d4110 = NULL`

## Uncertainty

`DAT_007d4110` vs `DAT_007d4120`: both appear to reference the D3D device. `0x007d4120` is set to 0 at start of teardown; `0x007d4110` is used for all vtable calls until `Release()`. These are 0x10 bytes apart. Possible: `0x007d4110` = IDirect3DDevice9* (the COM pointer), `0x007d4120` = a wrapped interface or HRESULT result slot.
