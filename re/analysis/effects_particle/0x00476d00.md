## Mechanical description
- Takes param_1 as a pointer to a pool-entry struct.
- Guards on `*(param_1+0xc) < *(param_1+8)` (write index < capacity); no-op if pool is full.
- If pool entry at +0x10 is non-null: writes the 3-dword global position (DAT_00692528/2c/30) into the position array at index `*(param_1+0xc)` with stride 0xc (3 floats × 4 bytes).
- If pool entry at +0x14 is non-null: writes the 2-dword global velocity (DAT_006924dc/e0) at stride 8 (2 floats).
- If pool entry at +0x18 is non-null: writes the 1-dword global colour/alpha (DAT_00692554) at stride 4.
- If pool entry at +0x1c is non-null: copies 0x10 dwords (64 bytes) from DAT_006924e8 at stride 0x40 (16 dwords = matrix-sized block). [UNCERTAIN U-0592]
- If pool entry at +0x20 is non-null: writes 1-dword scalar (DAT_006924d8) at stride 4.
- If pool entry at +0x24 is non-null: writes the 4-dword RGBA (DAT_00692598/9c/a0/a4) at stride 0x10.
- If pool entry at +0x28 is non-null: copies 8 dwords (32 bytes) from DAT_00692534 at stride 0x20.
- Increments `*(param_1+0xc)` (write index) by 1.
- This is the "append one particle to render batch" function; the pool struct carries separate arrays for each particle attribute.
