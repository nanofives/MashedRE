## Mechanical description
- Takes (int *param_1 [container {ptr,count,cap}], int param_2 [item pointer]).
- If count < cap: stores item at slot, refs (item+0x18 short), increments count. Returns old count (= index).
- Else: grow by realloc.
  - new_size = cap*4 + 0x50 (grows by 0x14 = 20 elements at a time = 80 bytes).
  - First call (ptr==0): vtable[+0x108] alloc with hint 0x1030008.
  - Else: vtable[+0x110] realloc.
  - On failure: error path with 0x80000013.
  - Updates ptr; cap += 0x14.
  - Stores item, refs, increments count.
- Returns slot index, or -1 on error.
- Container "append-with-refcount-bump" used for material/texture lists.

## Constants
- 0x18 — refcount offset on item.
- 0x50, 0x14 — grow amounts (bytes / elements).
- 0x108, 0x110 — vtable Alloc/Realloc.
- 0x1030008 — alloc hint (different from the 0x10502/0x30502 used in mesh code).
- 0x80000013 — error code.
