## Mechanical description
- Controller-config dialog initializer. param_1=HWND (in_EAX implicit per Ghidra).
- Calls `FUN_00499740()` (boot/save — SetControlTextFromResource per hooks.csv) on 9 specific control IDs:
  - 0x411 (string-id base + 0x7f8), 0x412 (+0x802), 0x413 (+0x80c), 0x414 (+0x816), 0x3ef..0x3f2 (all +0x820), 0x1 (+2000).
- Strings are indexed off `DAT_007719e8` (language base) + per-control offset.
- Iterates 4 control HWNDs (local_10[]) for player slots 0..3:
  - For each player, reads `(&DAT_007e96fc)[player*0x80]` (active flag — same slot as FUN_00497450/00497470):
    - If == 1 (joypad): `SendMessageA(hwnd, 0x143 (CB_ADDSTRING), 0, lparam=string ptr)` from `piVar5 - 0x4e` then `SendMessageA(hwnd, 0x151 (CB_SETITEMDATA), w=result, 1)`.
    - If == 2 (keyboard): `SendMessageA(hwnd, 0x143, 0, 0x7730b4)` (string addr) then `CB_SETITEMDATA(w, 2)`.
  - Always adds `0x7730f8` string with item-data 0 (None entry).
  - Sends `0x14e` (CB_SETCURSEL).
- 0x143=CB_ADDSTRING, 0x151=CB_SETITEMDATA, 0x14e=CB_SETCURSEL.
