## Mechanical description

Entry point 0x0045bba0 is a branch target within the body starting at 0x00453f60.

### Section 1 — Per-player activation loop

Iterates 4 player slots: `pfVar15 = &DAT_0088fc70`, stride 0x2d dwords (0xb4 bytes),
exits when `pfVar15 >= 0x88ff40` (4 iterations). `pfVar15` points to slot_base+0x90.

Guard: `FUN_0040e350() == 6` (race mode) AND `(&DAT_0068d1f0)[player_idx * 4] != 4`.

If slot active (slot+0xa8 != 0) AND countdown running (slot+0xac != 0):
1. Copies current position fields (derived from pfVar15 offsets) to start-position fields.
2. `pfStack_84 = FUN_0045bfe0()` — loads global Bezier track position 0x0080332c.
3. `FUN_004b4b60()` — targeting prerequisite check; only if non-zero → continue.
4. `FUN_0045c350(slot_ptr, param2)` — fire condition check (Bezier+path safety):
   - Returns 0 → `FUN_0045bac0()` + `FUN_00476880()` (deactivate + effect); abort.
   - Returns 1 → proceed to button check.
5. Re-checks slot+0xac != 0.
6. `(**(code **)(slot+0xa8 + 0xc))()` — vtable+0xc: CAN_FIRE query.
7. Reads controller button state:
   - `ctrl_idx = *(DAT_007f1a14 + player_idx*4) * 0x4c`
   - `cVar3 = (&DAT_007f103c)[ctrl_idx + 3]` — fire button byte
   - `(&DAT_007f14ff)[ctrl_idx]` — second fire flag

8. If `local_64 != 0`: `(**(code **)(slot+0xa8 + 8))()` — **FIRE** via vtable+8 on powerup type entry.
9. If `local_64 == 0` AND controller flags indicate release: `FUN_0045bac0()` (deactivate).

If slot+0xac == 0 (no active countdown) AND button `(&DAT_007f1055)[ctrl_idx] != 0`
AND `(&DAT_007f1515)[ctrl_idx] == 0`:
- `FUN_0045c010()` → `FUN_0045bfa0(slot_ptr)` — **PowerupSlotActivate** (picks up powerup).

### Section 2 — Per-type teardown pass

Iterates the powerup type table at `&PTR_FUN_005f99b4` (table entry+0x1c, stride 0x40),
calling each type's +0x1c function once. For DepthCharge this is `FUN_004540c0`.

### Section 3 — Per-type mode-specific dispatch

`FUN_00426c00()` returns game mode from `DAT_00644158`.
`puVar14 = &DAT_005f99d4` (= table TRUE base 0x005f9998 entry 0 at offset+0x3c).

- mode==2:    calls `*puVar14`   = entry+0x3c function (single-player specific)
- mode==0x21: calls `puVar14[-1]` = entry+0x38 function (network multiplayer specific)
Skips null entries. DepthCharge has mode-0x21 fn = LAB_00457af0 (inside unanalyzed fn).

### Section 4 — Deferred spawn countdown

Per entry: if counter > 0 → decrement; call `FUN_00484cf0` with stored args (spawns
delayed projectile with audio).

## Key finding: AI fire path

AI drivers fire powerups through the **same button-state read** in Section 1.
AI must write to `(&DAT_007f103c)[ctrl_idx*0x4c + 3]` and `(&DAT_007f14ff)[ctrl_idx*0x4c]`
to simulate a fire button press. The fire call `(*vtable+8)()` is identical for all players.
No separate AI-only fire dispatch was found inside the powerup subsystem.

The powerup **type entry** is stored at slot+0xa8; it is a pointer into the type table at
0x005f9998. The countdown at slot+0xac gates whether the per-frame fire check runs.
