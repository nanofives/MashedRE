## What it does

Drives the RWP37 physics world step and then syncs all vehicle RW frame transforms from/to game state.

### 1. Physics world guard

- Returns `false` immediately if `DAT_006CE274 == 0` (RW physics world not initialized).

### 2. Mode-dependent step counter

- `FUN_0040E350()`: if mode != 6 → clears `_DAT_006CAD48` step accumulator.

### 3. RWP37 world step

- When `DAT_0086CAA0 == 0x7B` (step counter threshold = 123):
  - `uVar5 = FUN_00426060()` — gets dt/state value.
  - If race mode 1 or 5..6: `FUN_00562460()` — mode-specific pre-step.
  - `FUN_0047D3C0(DAT_006CE274, uVar5)` — **RWP37 `RpWorldStepSimulation` call** (S-2626). Takes world ptr + dt.
  - `DAT_0086CAA0 = 0` — resets step counter.
- `DAT_0061331C` capped to 0.05f if over `_DAT_005CC9A0` (50ms dt cap).
- `_DAT_006CAD48 += 1`.

### 4. Game mode set

- `FUN_0040E340()` — secondary mode query; stores `_DAT_007F0F8C = iVar4 + 5000`.

### 5. Per-vehicle RW frame sync (4 player vehicles)

Loop: `local_8c = &DAT_00881F68`, stride 0x341 DWORDs, terminates at `0x885378`.

Per vehicle (if active: `local_8c[-0x26D] == 0.0`):
- `FUN_0046D4A0(&local_64, iVar4)` — gets vehicle transform matrix (returned by ptr).
- `FUN_0057C210((&DAT_006C9A78)[iVar4])` — gets RW frame handle.
- Gets physics body matrix ptr: `*(int *)(*(int *)(frame + 0x10) + 4)`.
- Gets current RW matrix: `*(int *)(*(int *)(frame + 0x10) + 8)`.
- **Position correction**: `(local_64[0xC/D/E] * _DAT_005CF014) + current_matrix_pos[C/D/E]`.
- **Angular correction**: `FUN_004233E0(angle, z)` — computes angle from position/velocity vectors (mapped).
- Computes angular delta: `fVar13 = atan2(local_90, ...) → wrapped to [-π, π]`.
- Builds correction matrix with zero cross-product + normalized angular delta.
- For modes 3 and 5: copies full matrix back to local_64 (applies corrected transform).
- `FUN_004C45F0()` — mark RW matrix dirty.  
- `FUN_004C0E50()` — sync RW frame.  
- `FUN_0055DFF0(frame, 0)` / `FUN_0055AC00(frame+0x24, frame, 1)` — visibility update.

### 6. Per-vehicle position write-back

Second loop: `pfVar9 = &DAT_008816D8`, stride 0x341, terminates at `0x884AE8`.

- `FUN_004C3DF0(&local_a8, &local_a8, 1, auStack_48)` — transforms position by matrix.
- Subtracts base position from matrix translation.
- `FUN_0046D4D0(vehicleIdx, matrix)` — applies updated matrix to vehicle rigid body (S-2630).

### 7. Return

- `uVar5 = FUN_0055DEB0(DAT_006CE274, DAT_0061331C)` — advance simulation.
- Returns `FUN_0047EA40(uVar5) != 0` (S-2627).
