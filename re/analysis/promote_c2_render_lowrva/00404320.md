## Mechanical description

Per-mode render dispatcher; reads `DAT_007f0fd0` (game-mode/screen selector) and branches to 4 mode arms (5, 8, 9, 10), each with the same prologue (EndUpdate + temp 0.8×0.8 view-window + BeginUpdate + set RW state 6,0 + 8,0) and epilogue (restore RW state 6,1 + 8,1 + EndUpdate + restore view-window + BeginUpdate).

Inside the prologue/epilogue brackets, query sub-state via `FUN_0040e350()` and dispatch:

| Mode | Sub-state 3/4/5 → | Sub-state 6 → |
|---|---|---|
| 9 | `FUN_00403d30()` | `FUN_004041c0()` (via fallthrough) |
| 8 | `FUN_00403db0()` | `FUN_004041c0()` (via fallthrough) |
| 5 | `FUN_00403ed0()` | `FUN_004041c0()` (via fallthrough) |
| 10 | `FUN_00403c60()` | per-entry loop over `&DAT_00636b88`..`0x636c00` (stride 20 bytes, 6 entries), calling `FUN_00403fa0(iVar4)` when `DAT_005d757c < (float)entry[4]`. iVar4 = per-entry index (0..N) |

The mode-10 epilogue is inlined separately (not via the `LAB_004043be` shared epilogue used by modes 5/8/9).

For modes 5/8/9 the prologue body:
1. `iVar4 = *DAT_007d3ff8` (camera ptr from RW globals[0]).
2. `FUN_004c19f0(iVar4)` — RwCameraEndUpdate.
3. `uStack_8 = *(undefined4*)(iVar4 + 0x68); uStack_4 = *(undefined4*)(iVar4 + 0x6c);` (save current view-window x/y).
4. `uStack_10 = uStack_c = 0x3f4ccccd` (write 0.8f × 2).
5. `FUN_004c1c80(iVar4, &uStack_10)` — SetViewWindow(0.8, 0.8).
6. `iVar1 = FUN_004c1a00(iVar4)` — BeginUpdate; early-return if `iVar1 == 0`.
7. `(*(code*)DAT_007d3ff8[8])(6, 0)` — RW state slot 8 set RS#6.
8. `(*(code*)DAT_007d3ff8[8])(8, 0)` — RW state slot 8 set RS#8.

Sub-state query is `FUN_0040e350()` re-invoked up to 3× per mode (short-circuit chain checks for `iVar1 == 5 || iVar1 == 4 || iVar1 == 3`); on match calls per-mode renderer and jumps to `LAB_004043be` shared epilogue. Otherwise re-queries `FUN_0040e350()`; on `iVar1 == 6` calls `FUN_004041c0()`.

The shared epilogue at `LAB_004043be`:
1. `(*(code*)DAT_007d3ff8[8])(6, 1)` — restore RS#6.
2. `(*(code*)DAT_007d3ff8[8])(8, 1)` — restore RS#8.
3. `FUN_004c19f0(iVar4)` — EndUpdate.
4. `FUN_004c1c80(iVar4, &uStack_8)` — restore view-window from saved [0x68/0x6c].
5. `FUN_004c1a00(iVar4)` — BeginUpdate (resume normal).
