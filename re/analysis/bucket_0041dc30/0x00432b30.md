## Mechanical description
- Massive HUD-glyph-pair emitter for race control prompts. param_1 (in_EAX) = mode (0/2/other = current page), param_2 = filter-flag (used for prev-frame compare).
- Resolves two table lookups via FUN_0042add0 (current mode lookup) — assigns "iVar3" (current button-id).
- 7-case switch on param_1 (1/2/4/5/6/8/10) — each case calls FUN_0042ad10 to emit a glyph at (0x40, 0x1ac) coords with primary glyph code (e.g. 0x42, 0x43, 0x225, 0x48+0x13, 0x58, FUN_0042a9c0 result, 0x133).
- Each outer case has inner switch on iVar3 with same 7 codes for the second glyph (modifier glyph). Combination produces "press X to do Y" hotkey display.
- Special cases (FUN_0042b920 == 0x16): for case 1 + iVar3 mismatch, marks second glyph as 0xffffffff (no-display).
- Other constants written to slot fields: (+0x10) = 0xffffffff/0x1000/0x1, (+0x30) = 0x1ff/0x0.
- Conceptually: button-prompt HUD selector ("Press [A] to confirm" etc.) wired to current race mode.
