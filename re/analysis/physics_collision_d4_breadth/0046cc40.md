0x00468d80). Stride: `param_2 += 0x24` (36 floats = 144 bytes) per outer iteration.

For each terrain entry reads 12 floats at `param_2[0..0xb]`:
- `param_2[0..2]`  — triangle vertex A (x,y,z)
- `param_2[5..7]`  — triangle vertex B (x,y,z)
- `param_2[8..10]` — triangle vertex C (x,y,z)
- `param_2[9..11]` — unit face normal (fVar11, fVar12, fVar13)

Inner loop: 4 wheel slots.
- Transformed wheel positions: `DAT_0088e624` (float[3] per slot, stride 12 bytes)
- Per-slot "skip" flag: `DAT_0088e5e0[slot]` — skips slot if `!= 0`

**Depth test** (using vertex C + normal):
```c
depth = (wheel.x − param_2[8])*fVar11 + (wheel.y − param_2[9])*fVar12
       + (wheel.z − param_2[10])*fVar13;
```


1. Calls `FUN_00468b40(param_2, param_1)` — VehicleContactHistoryLookup (returns 0=new / 1=already active).
2. If new contact: writes into wheel slot at `vehicle + 0x65 + slot*0x31` (local_90):
   - `local_90[0x1b..0x1d]` = `param_2[9..11]` (face normal xyz)
   - `local_90[0]` = depth
   - `local_90[0x16..0x17]` = `param_2[0xc..0xd]` (surface metadata — [UNCERTAIN U-3630])

3. Debug event type=6 enqueued to ring `DAT_007e9de4` / index `DAT_007e9de0`:
   - White (0xff010101): sentinel -1.7147562e+38 contact (first-frame).
   - Blue  (0xff0000ff): sentinel -1.7014636e+38 contact (persistent).
   - Additional blue event when `param_1[0x27c] == 1` and `param_1[0x27d] > 0x32` and
     terrain type not -NAN.

Early return when `DAT_0088e650 == 4` (4 wheel contacts filled).
