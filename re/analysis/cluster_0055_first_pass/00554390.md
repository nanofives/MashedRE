object or NULL.

1. Alloc font object via FUN_005551d0().
2. Read 4 bytes (`local_2c`) — header word; abort if (local_2c >> 24)

3. Read 7 metadata dwords: piVar2[0], [1], [2], [3], [4] (likely

   (a count and another count, e.g. table-size & glyph-count).
4. Read `iVar6 = piVar2[0x4a]` (size of dispatch-table?). If > 0:
   - Alloc 2*iVar6 bytes (UTF tail-byte map?), zero-fill, store at
     piVar2[0x4b], read 2*iVar6 bytes from stream.

   256 entries × 2 bytes = 0x200... wait, says 0x100 read into
   piVar2+9 which is at +0x24, so this is 256 shorts of glyph idx).
6. Alloc piVar2[0x4c] * 0x20 byte glyph table at piVar2[0x4d], zero
   it. Then either:
   - If `*piVar2 == 0` (mode 0 — uncompressed glyphs):
     For each glyph (piVar2[0x4c] iterations): read 0x10 bytes

     resource and store at piVar6[0]. Set piVar2[0x4e] = LAB_00554940
     (uncompressed-mode glyph fetcher).

     For each glyph: read 4 bytes header, read 4 bytes id; if

## Subsystem hypothesis

**Vector-font binary loader**. Mashed appears to ship glyphs as
small (moveto/lineto/curveto) path commands. The two modes
(uncompressed-shared-paths vs embedded-paths-per-glyph) match
"TTF-like" vs "small-bitmap" alternatives. ASCII-range fast lookup
(piVar2[9..0x49]) + Unicode tail-byte map (piVar2[0x4b]) confirms
multilingual support.

Caller FUN_00427880 is in the boot/launch band — fonts loaded
during start-up, matches the typical engine layout.
