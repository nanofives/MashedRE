## Mechanical description

**Vector-font text-format loader**. Reads a text file via the RtFS
file handle `param_4`, parsing line-by-line into a font object
`param_1`.

1. Init font header: param_1[0]=1, [1]=1.0f, [2..3]=0.
2. Read first line (FUN_00550a20 = RtFile_gets).
3. Default glyph table = 0x100; alloc array at param_1[0x4d].
4. Try-parse FIRST line via vtable+0x104 (sscanf-style) format
   `s_______00623f58`. If matches (return 1): enter the main loop.
5. Loop while `FUN_00550a20` returns non-zero (next line):
   - If no glyph in flight (`uStack_b8 == 0`): decode the line as a
     UTF-8/16/32 character code (single-byte ASCII, 2/3/4/5/6-byte
     extended sequences via 0xC0/E0/F0/F8/FC mask test). On valid
     char < 0x10000, fall through to LAB_00556c34 (continue).
   - Else (glyph in flight): match line against:
     - `s_begin` (0x00623f50) → FUN_00553ef0 + FUN_00553f40 (start a
       new path sub-list).
     - `0x00623f48` (closepath "end" or similar) → FUN_00553e00 +
       FUN_00554010 (materialize), then register glyph: update
       fast/slow lookup tables based on glyph_id < 0x80, copy first
       path ptr into glyph-array; grow glyph-array by 0x20 when full.
     - `s_moveto %f %f` (0x00623f38) → FUN_00553030 with parsed (x,y).
     - `s_lineto %f %f` (0x00623f28) → FUN_005530f0.
     - `s_curveto %f %f %f %f %f %f` (0x00623f0c) → FUN_005531f0
       (3-pt cubic).
     - `s_closepath` (0x00623f00) → FUN_005533d0.
6. After EOF: store final glyph count, call FUN_00555f20 (post-
   process / build dispatch tables), patch the vtable[0x4e] to
   LAB_005547f0 (compressed-mode renderer).

## Subsystem hypothesis

**Plaintext vector-font loader**. This is the smoking-gun confirmation
that Mashed has its own custom vector-font format with begin/end
glyph blocks and SVG-like (moveto/lineto/curveto/closepath) path
commands. Companion to the BINARY loader FUN_00554390 — likely used
for dev/debug; the binary form is shipped.
