## Mechanical description

**Sorted-key insertion** into a parallel-array map (`__thiscall(this, key, out_index)`). Maintains two arrays kept in lockstep:
- `*(this+0x18)` = key array (uint*)
- `*(this+0x1c)` = value array (uint*)
- `*(this+0x14)` = count

### Flow

1. **Binary search** for `param_2 = key`:
   - lo=0, hi=count, mid=count>>1.
   - Loop: read `key_at_mid = *(keys + mid*4)`. If `< param_2`: lo=mid+1. If `> param_2`: hi=mid. If `==`: break with `local_8=mid`.
   - On exit, `local_8` is either the insertion point or the matching index.
2. **If exact match found (uVar4 <= uVar8)**: write `out_index = local_8`, return 0 (already present).
3. **Otherwise insert at `local_8`:**
   - **Grow arrays if at power-of-two capacity** (`count == (-count & count)`):
     - new_cap = (count == 0 ? 1 : count*2).
     - `operator_new(new_cap << 2)` for keys; copy + free old. Same for values.
     - On alloc failure: return `0x8007000e`.
   - **Shift right** (entries `local_8`..count-1 → `local_8+1`..count) — unrolled dword/byte loop.
   - Write `keys[local_8] = param_2`, `values[local_8] = 1` (default mode).
   - `count++`.
4. Write `out_index = local_8` if non-NULL.

This is exactly the inner of the `#pragma warning` registry: each `(warning_id, mode)` pair gets inserted in sorted order so binary-search lookup is O(log n).
