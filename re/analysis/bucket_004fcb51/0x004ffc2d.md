## Mechanical description

**Advance source cursor to end-of-line, return 1 if `\n` was found, 0 if hit end-of-source** (`__fastcall(this)`).

Operates on the lexer-input state (cursor at `*this`, end at `this[1]`, line counter at `this[7]`):

- Loop while `*this < end`:
  - Peek `*cursor`:
    - `'\n'`: leave cursor on the LF, return 1.
    - `'\\'`:
      - If next char is `'\n'` (Unix backslash-newline): advance cursor by 2, increment `this[7]`.
      - Else if next two chars are `'\r' '\n'` (Windows): advance by 3, increment `this[7]`.
      - Else: treat backslash as ordinary, advance by 1.
    - Else: advance by 1.
- Return 0 (no LF before EOS).

This is the standard "skip to next newline, respecting line-continuation" preprocessor primitive — used by all "directive: ignore rest of line" handlers and by `#error`'s message-extractor.
