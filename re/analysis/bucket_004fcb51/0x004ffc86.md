## Mechanical description

**Floating-point literal lexer** (`__thiscall(this, source_ptr, out_double_or_0)`).

Recognizes the C/HLSL floating-point grammar from the cursor pointed by `param_2`:

```
digits '.' digits? ('e' '-'? digits)?
'.' digits ('e' '-'? digits)?
```

Returns the number of bytes consumed (0 if not a valid float).

### Flow

1. **Integer part scan** (only if first char `isdigit`):
   - Advance while `isdigit`.
   - Expect `'.'`. If missing → return 0.
   - Advance past `.` then scan fractional digits.
2. **Fraction-first form** (`.123`):
   - First char must be `.` AND next char must be digit.
   - Advance past `.` then scan fractional digits.
3. **Optional exponent**:
   - Check `tolower(*cursor) == 'e'`.
   - If next is digit: scan digits.
   - Else if next is `'-'` and the one after is digit: skip `-`, scan digits.
4. **Convert** (only if `out_double_or_0 != 0`):
   - Use `__chkstk` to reserve a stack frame.
   - Copy `[param_2..cursor)` into the stack via unrolled memcpy.
   - NUL-terminate.
   - Call `_atof(stack_buf)` → store at `*out_double_or_0`.
5. Return `cursor - param_2` (bytes consumed).

The lookup `*this[1]` (which is the lexer-input-state's end-pointer) gates every advance, ensuring no OOB reads. The `param_1` here matches the FUN_004ffbc5 / FUN_004ffbab state object (offset +4 is "end").
