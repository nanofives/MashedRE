## Mechanical description

**Hex literal lexer** (`__thiscall(this, source_ptr, out_uint_or_null)`).

Recognises `0x[0-9a-fA-F]+` literals, parses to a 32-bit unsigned integer, emits a truncation warning if more than 8 hex digits were seen (= 32 bits worth), and returns the number of source bytes consumed.

### Flow

1. End-pointer guard: `*(this+4) <= param_2 + 2` AND `*param_2 != '0'` AND `param_2[1] != 'x'` AND `_isxdigit(param_2[2]) == 0` → return 0 (not a hex prefix).
2. Initialise `iVar2 = 0`; advance `pcVar4 = param_2 + 2`.
3. While `pcVar4 < end` AND `_isxdigit(*pcVar4)`:
   - `iVar2 *= 0x10`.
   - Decode digit:
     - `cVar1 >= 'a'`: `iVar2 += cVar1 - 0x57` (= `cVar1 - 'a' + 10`).
     - `cVar1 >= 'A'`: `iVar2 += cVar1 - 0x37` (= `cVar1 - 'A' + 10`).
     - else: `iVar2 += cVar1 - 0x30` (= `cVar1 - '0'`).
   - Advance.
4. Write `*param_3 = iVar2` (if non-null).
5. `iVar2 = pcVar4 - param_2` (= total bytes including `0x` prefix).
6. If `iVar2 > 10` (i.e. more than 8 hex digits after `0x`): emit warning code 0x3ea `"hex value truncated to 32bits"` via FUN_005000c2.
7. Return `iVar2`.

### Notes on prefix check

The guard `param_2 + 2 < *(this+4) && *param_2 == '0' && param_2[1] == 'x' && _isxdigit(param_2[2])` requires at least one hex digit to follow `0x`. A bare `"0x"` returns 0.

The hex-decode uses MSVC's classical `'a'-10=0x57, 'A'-10=0x37, '0'=0x30` offset trio.
