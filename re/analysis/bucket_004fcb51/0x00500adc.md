## Mechanical description

**Main lexer dispatch / raw-token producer** (`__thiscall(this, mode_mask, out_token)`).

The central "give me the next raw token" routine for the shader-compiler lexer. Stores the caller's mode mask, snapshots the source-location, skips whitespace + comments via `FUN_0050042f`, then probes each token recogniser in the fixed precedence order below, writing the token-class into `*out_token` and returning 0 to the caller (token data lives in `out_token[0..7]`). This is the recogniser-chain anchor referenced as `FUN_00500adc` by 8 different production-level callers, and the function FUN_004fd0d3 (the preprocessor) opens it as `*(this+0x80) | 4`.

### Token-record layout (`param_3` = `out_token`)

| `out_token` slot | Meaning |
|---|---|
| `[0]` | token-class (see table below) |
| `[1]` | payload value (uint, char, etc.) |
| `[2]`/`[3]` | per-recogniser payload (e.g. strdup pointer, double low/high) |
| `[4]` | source line at start of token (snapshot of `*(this+0x18)` = `param_1[6]`) |
| `[5]` | source line at start of skip-pass (snapshot of `*(this+0x1c)` = `param_1[7]`) |
| `[6]` | source-cursor at start of token (= `*this` before consumption) |
| `[7]` | bytes consumed |

### Dispatch precedence

The recognisers are tried in this fixed order. The first one that returns non-zero "wins". Each writes `*out_token = class`.

| Order | Class written | Recogniser | Token | Cross-ref |
|---|---|---|---|---|
| 0 | 0xc | (n/a) | "whitespace skip hit EOF" | from `FUN_0050042f` returning 1 |
| 1 | 0xd | (n/a) | "EOF after skip" | `*this >= *(this+4)` |
| 2 | 5 | `FUN_004ffc86` | float literal | (then `FUN_004fff8d` for `f`/`h` suffix; bumps class 5→6/7) |
| 3 | 2 | `FUN_005009ba` | char literal `'X'` | (class fixed at 2, integer) |

### Flow

1. `param_1[10] = param_2` — stash the caller's mode mask into the lexer-input-state's `flags` slot. This is what FUN_0050042f reads as `param_1[10] & 2` (ASM mode) and what the recognisers read at `*(this+0x28)` (escape, include-mode, profile-prefix bits).
2. Snapshot `out_token[4]/[5]` ← line/sub-line numbers (so post-decode we report the line at token start).
3. `iVar1 = FUN_0050042f()` — whitespace/comment skip.
   - Returns 1 → `*out_token = 0xc` (whitespace-EOF sentinel), bail out via LAB_00500c36.
4. Re-snapshot `out_token[4]/[5]` ← line/sub-line after skip (file may have changed due to `#include` push in caller stack).
5. `*this >= *(this+4)`: emit `*out_token = 0xd` (EOF sentinel) and goto LAB_00500c36.
6. Otherwise: run dispatch chain (above). On first hit:
   - Compute `iVar2 = bytes_consumed`.
   - For float and integer hits, run the suffix probe (`FUN_004fff8d` after float, `FUN_004fffd3` after integer) and add its consumed bytes to `iVar2`.
7. LAB_00500c36: write `out_token[6] = *this` (token start cursor), `out_token[7] = iVar2` (consumed bytes), advance `*this += iVar2`.
8. Return 0.

### Cross-reference

This routine is the input side of the preprocessor `FUN_004fd0d3` ("preprocessor token classifier"). The mode-mask bit 4 ORed in by FUN_004fd0d3 (`*(this+0x80) | 4`) is the "disable escapes" flag that FUN_00500721 / FUN_005009f9 read from `*(this+0x28) & 4`. Other bits used:
- `0x02` → enable `;`-comments (ASM mode) in FUN_0050042f; enable shader-profile prefix lexer in FUN_005008b7.
- `0x04` → disable backslash escapes (set during preprocessor scan).
- `0x08` → enable `<...>` include-string mode in FUN_005009f9.
