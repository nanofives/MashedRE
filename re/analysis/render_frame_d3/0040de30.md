## Analysis

Configures the overlay/minimap camera for orthographic rendering, renders the minimap world geometry, then leaves the camera in the modified state (restoration is done by `FUN_0040df20`).

Three-phase pattern:
1. **Save**: copies frame matrix (16 DWORDs from frame+0x10 = modelling matrix), saves view window X/Y (`camera+0x68/0x6c`), saves projection (`camera+0x14`).
2. **Override**: applies identity transform to frame via `FUN_004c1480`; sets orthographic view window 0.6 × 0.45 and projection type 2 (`rwPARALLEL`).
3. **Render**: calls `FUN_00426670` (WorldRenderDispatch_Begin) with the reconfigured camera.

`camera+0x14` = projection type in the RwCamera layout (observed at this offset; confirmed against known `RwCameraSetProjection` write pattern).

`FUN_004c1480`: likely `RwFrameTransform` or `RwFrameSetIdentity`-variant (takes frame, matrix, flags). [UNCERTAIN U-1709: exact identity — is flags arg 0 = overwrite mode?]

The 0.6/0.45 view window ratio gives aspect ≈ 0.75 (3:4), consistent with a minimap viewport that occupies a portion of a 4:3 screen.

Save buffer at `DAT_0063b928` (matrix), `DAT_0063b9ac/b0` (view window), `DAT_0063b9a8` (projection) is shared with `FUN_0040df20` (restore side).
