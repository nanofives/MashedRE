## Mechanical description
- **Remove primitive `param_2` from the broadphase octree** (counterpart to the INSERT at FUN_00564c80).
- First calls `FUN_00563f60(*(this+0xc010), param_2)` — removes all pair entries referencing this primitive from the pair-manager (this is the "broadphase pair table" the insert registered into).
- Reads location record `slot = this[0x8020 + prim*2]` (per-prim 2-byte location tag — a different table from the AABB stride).
  - Low 6 bits = octant index (0..7), 0x3f = "in root chain", 0xffff = "in root chain (sentinel)".
  - High 10 bits (>>6) = parent node id.
- TWO CHAIN-REMOVE PATHS:
  - Root-chain path (`octant == 0x3f || 0xffff`): scans the linked list at `this + (node*5 + 0x2607)*4` head, follows chain successors at `0x8820 + idx*4`, unlinks prim's record by patching predecessor's chain-next.
  - Octant-leaf path (`octant in 0..7`): scans the chain at `this + 0x9820 + ((octant + node*10)*2)` low 10 bits, follows the same 0x8820 chain-successor table, unlinks via predecessor write.
- After unlink: pushes the freed record onto the global free list at `this + 0x981a` (record_free_head, matches FUN_00564c80).
- Optional leaf-count decrement: if removed entry had the "primitive" flag (high bit `0x80` at byte +3 of 0x881f record), decrement the leaf's primitive-count field by 0x400. If the count was saturated (`0x7c00`), recompute by walking up to 0x1f primitives in the chain to find the actual count.
- **Merge phase**: if `node != 0` (not root) AND parent's leaf-record is now empty (low 10 bits == 0x3ff), walks the 8 octant slots of this parent: if any sibling octant still has primitives OR is not a leaf-flag, abort merge. Otherwise: marks parent's slot in grand-parent as empty (`0x83ff`), reclaims the internal node into the free chain at `this + 0xc00e` (matches FUN_00564c80 split allocator).
- This is the canonical octree-remove with bottom-up empty-node collapse.
