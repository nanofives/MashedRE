## Mechanical description

Bounds-check: if `param_1 >= 16`, returns 0.

Computes slot offset `iVar2 = param_1 * 0xD04`. Writes 24 float values into a large per-slot table at `DAT_0x00881630 + iVar2`.

Phase 1 — copy input box corners (6 floats from `param_2[0..5]`) into 8 vertex slots (12 uint32 slots at `+0x00..+0x2C`):
- Writes an arrangement of `param_2[0..5]` at offsets 0x00..0x2C, replicating each component across the 8 box corners.

Phase 2 — compute edge midpoints (multiply each pair sum by `DAT_0x005cc32c` ≈ 0.5):
- Offsets 0x60..0x8C: midpoints of opposite corner pairs.

Phase 3 — compute face centroids (multiply triple sums by `DAT_0x005ce034` ≈ 0.333):
- Offsets 0x6C..0xB4: averages of face-adjacent triplets.

Phase 4 — additional corner-to-midpoint averages:
- Offsets 0x84..0x8C.

Returns 1.
