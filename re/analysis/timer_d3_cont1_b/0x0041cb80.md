## Reimpl note — non-standard ABI (register-passing callee, ESI-arg)

Verified via disassembly (0x0041cb80 body):
```
0x0041cb80  PUSH ESI                  ; save callee-saved ESI
0x0041cb81  CALL 0x0040e340
0x0041cb86  MOV  [0x0063cde4], EAX    ; store ret of FUN_0040e340
0x0041cb8b  CALL 0x0040b890
0x0041cb90  MOV  [0x0063ce18], EAX    ; store ret of FUN_0040b890
0x0041cb95  MOV  ESI, 0x0063ce20      ; loop base
0x0041cb9a  LEA  EBX, [EBX]           ; 6-byte NOP for alignment
0x0041cba0  CALL 0x0041c380           ; <-- ESI carries per-element ptr
0x0041cba5  ADD  ESI, 0x114
0x0041cbab  CMP  ESI, 0x0063d270
0x0041cbb1  JL   0x0041cba0
0x0041cbb3  POP  ESI
0x0041cbb4  RET
```

FUN_0041c380's first instruction is `MOV ECX, dword ptr [ESI + 0x108]` —
the callee takes its per-element pointer in **ESI**, not EAX, not on stack.
ESI is preserved as a callee-saved register across the CALL — that's the
whole mechanism: each iteration the loop simply re-uses live ESI.

A plain C++ function-pointer call (`callee()`) cannot set ESI deterministically:
under MSVC the compiler is free to clobber ESI before our `callee()` call site,
and even if it didn't, the C++ source has no way to express "leave ESI = ptr".
This is the same impedance class as 0x00422120 TimerInitLoop (EAX-arg) — both
need an asm-level fix.

**Fix pattern (apply in next session):** rewrite as `__declspec(naked)` +
inline-asm that replicates the original prologue / loop body / epilogue
byte-for-byte.  Canonical template: `TimerInitLoop` in
`mashedmod/src/mashed_re/Util/TimerInit.cpp` (feature/timer-init-naked,
frida-sweep-20260517-2121+).

Differences from the TimerInitLoop template for 0x0041cb80:
  - Two **stack-arg** prologue calls (`call 0x0040e340`, `call 0x0040b890`)
    happen before the loop and store EAX returns to globals.  These can be
    folded into the naked body as `mov ecx, 0x0040e340; call ecx;
    mov [0x0063cde4], eax` — keeps everything in one __asm block.
  - The per-element loop passes the pointer in **ESI** (not EAX as in
    TimerInitLoop).  The asm body is therefore:
        mov esi, 0x0063ce20
      loop_body:
        mov ecx, 0x0041c380
        call ecx                    ; ESI is the implicit arg, preserved
        add esi, 0x114
        cmp esi, 0x0063d270
        jl  loop_body
  - There is a 6-byte LEA-as-NOP at 0x0041cb9a for alignment — purely
    cosmetic; the reimpl does not need to replicate it.
