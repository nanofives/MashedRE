## Mechanical description

Initializes physics/suspension parameters for vehicle slot `(uint)param_1` (0-15). If `(uint)param_1 >= 16`, returns 0.

Slot offset: `iVar5 = (int)param_1 * 0xD04`.

### Phase 1 — default spring/mass constants

Writes defaults to locals (overridden below):
- `DAT_0x006131**: spring force = 100.0, spring limit = 40000.0, stiffness = 1.0, damping = 1.5

### Phase 2 — lookup override from config table

Calls `FUN_00430790()` (country code), `FUN_0042f6a0()` (game mode). If mode == 10, calls `FUN_00431d70()` (course leader index). Then `FUN_0040ce80(uVar4)` → track type int `iVar5`.

Iterates a config table at `DAT_0x00613140/0x00613148` (terminated by -1 sentinel). If entry key matches track type, overrides the 4 spring constants with scaled values.

### Phase 3 — write per-slot suspension geometry

Writes to `DAT_0x008815a4 + slot * 0x341` (per-slot flag table):
- Flags at offsets 0x00 = 1, 0x0C = 0.
- Float fields: `DAT_008816f0..DAT_008816f8`: suspension geometry floats (0x3e19999a ≈ 0.15, 0x3fa51eb8 ≈ 1.29).
- Many other float slots for wheel positions, suspension arm angles, spring rates.

### Phase 4 — FastSqrt-based radius computation

Iterates 4 wheel records (stride 0x31 uint32 = 0xC4 bytes from `pfVar8 = DAT_008817 10 + iVar5`):
- Computes `sqrt(x²+y²+z²)` using `FUN_004c3b30` (FastSqrt, C4).
- Tracks maximum radius → writes to `DAT_00881a40 + iVar5`.

### Phase 5 — distance-from-center per wheel/point

Further loops computing distances from `DAT_008816fc/700/704` center to wheel attachment points, multiplied by constant from `s_Afff_SandSpWheel_005cc753._1_4_` (sand-surface wheel parameter string embedded in constant pool).

### Phase 6 — spring force distribution

Computes total spring force `fVar1` as sum of 4 wheel spring values. Distributes mass across wheels via reciprocal weighting:
- `DAT_008815f8 + iVar5` = distributed force sum.
- `DAT_008815fc + iVar5` = `1.0 / fVar1`.

Returns 1.
