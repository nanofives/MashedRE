# STEP 3 — DESIGN THE NEW exe_main.cpp
The new exe_main.cpp structure:

```cpp
// Mashed RE -- standalone executable.
// Milestone A: opens a real Win32 window that drains its message pump.
// No menu rendering yet (that's milestone B).
#include <windows.h>

// Forward-decls for the C3+ Boot functions we'll call.
// These are exported from the Boot/*.cpp files compiled into mashed_re.exe.
namespace mashed_re::Boot {
    bool ParseCommandLine(LPSTR cmdline);  // wraps 0x00493900 logic
    bool LoadVideoConfig();                 // wraps video-config load
    bool CreateMainWindow(HINSTANCE);       // wraps 0x00499ba0
    bool ShowMainWindow();                  // wraps 0x004996f0
    bool InitRwEngine();                    // wraps 0x00493710 + 00493600 + 00493640
    void DestroyMainWindow();               // wraps 0x00499cc0
    void ReleaseCom();                      // wraps 0x00494bc0
    bool MainLoopInit();                    // wraps 0x00492770
    bool PumpMessages();                    // wraps 0x00499690 (returns true if quit)
    void FrameTick();                       // wraps 0x004926c0 (QPC frame tick)
    void FrameDispatch();                   // wraps 0x00492e90 (per-frame game tick)
}

int WINAPI WinMain(HINSTANCE hInstance, HINSTANCE, LPSTR lpCmdLine, int) {
    using namespace mashed_re::Boot;

    if (!ParseCommandLine(lpCmdLine)) return 1;
    if (!LoadVideoConfig())           return 2;
    if (!CreateMainWindow(hInstance)) return 3;
    if (!ShowMainWindow())            return 4;
    if (!InitRwEngine())              return 5;
    if (!MainLoopInit())              return 6;

    // Main loop. For milestone A, we just drain messages and tick the frame;
    // FrameDispatch will execute the game's per-frame logic against whatever
    // global state has been initialized so far. Most of that state is null/zero
    // because we haven't loaded assets, so FrameDispatch may early-return.
    // That's fine -- the window will still be blank but visible.
    while (!PumpMessages()) {
        FrameTick();
        FrameDispatch();
    }

    // Teardown.
    DestroyMainWindow();
    ReleaseCom();
    return 0;
}
```

The exact function names in the namespace will depend on what the C3+ Boot
files actually export. Map them to the structure above. If a function doesn't
exist yet, leave a `// TODO: not-yet-promoted` stub.
